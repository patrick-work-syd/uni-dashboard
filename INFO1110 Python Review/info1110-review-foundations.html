Variables, Types & Expressions

Lessons 1–2: Variables as labels, basic types, arithmetic
›

In Python, a variable is a label that points to an object in memory. It doesn't declare a type — the type is determined by the object it refers to.

# Variables as labels on objects
name = "Alice"         # points to a string object
age = 20             # points to an integer object
score = 95.5         # points to a float object
is_student = True   # points to a boolean object

# Dynamic typing — same variable can point to different types
x = 5                # integer
x = "hello"      # now it's a string (type changed!)

# Arithmetic operators
result = 10 + 5      # 15
result = 10 - 5      # 5
result = 10 * 5      # 50
result = 10 / 5      # 2.0 (always a float in Python 3)
result = 10 // 5     # 2 (integer division)
result = 10 % 3      # 1 (modulo — remainder)

# String formatting with f-strings
message = f"Hello, {name}! You are {age} years old."

Python

x = 5
x = "hello"  # OK — type changes

Java

int x = 5;
// x = "hello";  // ERROR — type mismatch!

Key difference: Java requires you to declare the type upfront. Once x is declared as an int, it can only hold integers. Python is more flexible — the same variable can hold any type.

🌉
Java parallel: In Java, you write int age = 20; or String name = "Alice";. The type comes first, then the variable name. Java's types are primitive (int, float, boolean) or object (String, ArrayList). String formatting in Java uses System.out.println("Hello, " + name) or String.format().
✎ Quick check
What is the main difference between Python's dynamic typing and Java's static typing?
Python: Variables have no declared type; the type is determined by the object they point to. The same variable can switch types at runtime. Java: Every variable must have a declared type, set when you create the variable. Once declared, it can only hold values of that type. Java's type system is enforced at compile time; Python's is flexible at runtime.

Input/Output & String Operations

Lesson 2–3: print(), input(), string methods
›

Python's input() reads user input as a string. print() displays output. Strings have built-in methods like .upper(), .lower(), .strip(), .split().

# Reading input (always returns a string)
name = input("What is your name? ")

# Output
print("Hello, " + name)

# String methods
text = "  Hello World  "
print(text.strip())        # "Hello World" (removes whitespace)
print(text.upper())        # "  HELLO WORLD  "
print(text.lower())        # "  hello world  "

words = "apple,banana,cherry"
print(words.split(","))  # ['apple', 'banana', 'cherry']

Python

name = input("Name? ")
print(name.upper())

Java

Scanner sc = new Scanner(System.in);
String name = sc.nextLine();
System.out.println(name.toUpperCase());

String methods in Java are very similar: .toUpperCase(), .toLowerCase(), .trim(), .split(). The main difference is Java requires explicit imports for input (Scanner) and uses System.out.println() instead of print().

🌉
Java parallel: Java uses Scanner for input and System.out.println() for output. String methods exist in Java but with different names (camelCase instead of snake_case). Both languages support string concatenation with +.
✎ Quick check
What does input() always return in Python, and why would you need to convert it if you want a number?
input() always returns a string, even if the user types a number like "42". To convert it to an integer or float, you use int(), float(), etc. For example: age = int(input("Age? ")). This is why you need explicit conversion — Python doesn't assume what type you want.

Conditionals: if/elif/else

Lesson 3: Boolean expressions, comparison, logical operators
›

Conditionals let you run different code based on conditions. Python uses if, elif, else. Indentation matters!

# Comparison operators
x = 10
print(x == 5)   # False (equal)
print(x != 5)   # True (not equal)
print(x > 5)    # True (greater than)
print(x <= 5)   # False (less than or equal)

# Logical operators: and, or, not
age = 20
print(age >= 18 and age < 65)  # True (both conditions true)
print(age < 13 or age >= 65)   # False (neither true)
print(not (age > 30))          # True (negation)

# if/elif/else chains
score = 85
if score >= 90:
    print("A")
elif score >= 80:
    print("B")
elif score >= 70:
    print("C")
else:
    print("F")

Python

if x > 5:
    print("big")
else:
    print("small")

Java

if (x > 5) {
    System.out.println("big");
} else {
    System.out.println("small");
}

Logic is identical. Syntax differences: Java uses braces {} instead of indentation. Logical operators: Java uses && (and), || (or), ! (not).

🌉
Java parallel: Java requires parentheses around conditions: if (x > 5) not if x > 5. Braces define code blocks instead of indentation. Operators: ==, !=, >, < work the same. But use &&, ||, ! instead of and, or, not.
✎ Quick check
What is the order of evaluation for a or b and c? Which operator runs first?
and has higher precedence than or, so the expression evaluates as a or (b and c). This is the same in Java: || has lower precedence than &&. Parentheses make this explicit: a || (b && c).

Loops: for and while

Lesson 4: Iteration, break, continue, loop tracing
›

for loops iterate over sequences. while loops run while a condition is true. Both support break (exit loop) and continue (skip to next iteration).

# for loop — iterate over items
for item in [1, 2, 3]:
    print(item)

# for loop with range
for i in range(5):  # 0, 1, 2, 3, 4
    print(i)

# while loop
count = 0
while count < 5:
    print(count)
    count += 1

# break and continue
for i in range(10):
    if i == 3:
        continue  # skip 3, go to next iteration
    if i == 7:
        break     # exit loop entirely
    print(i)

Python

for i in range(3):
    print(i)

Java

for (int i = 0; i < 3; i++) {
    System.out.println(i);
}

Java's for loop syntax is different. It declares the counter variable, sets the condition, and specifies how to increment. But while loops are identical in both languages.

🌉
Java parallel: Java has two main loop styles. (1) Traditional: for (int i = 0; i < 10; i++) — declare, condition, increment. (2) Enhanced: for (int item : list) — iterate over items (same as Python's for item in list). Both break and continue work identically.
✎ Quick check
What would Java code that iterates over a list look like? (The enhanced for loop version.)
for (String item : list) {
    System.out.println(item);
}
This is Java's enhanced for loop (also called for-each). The syntax is for (Type variable : collection) — very similar to Python's for item in list.

Lists & Tuples

Lesson 5: Creating, indexing, slicing, methods
›

Lists are mutable (changeable) sequences. Tuples are immutable (fixed). Both support indexing and slicing.

# Creating lists
numbers = [1, 2, 3, 4]
names = ["Alice", "Bob", "Charlie"]
mixed = [1, "hello", 3.14]  # lists can mix types

# Indexing and slicing
print(numbers[0])      # 1 (first element, 0-indexed)
print(numbers[-1])     # 4 (last element)
print(numbers[1:3])    # [2, 3] (slice: index 1 up to but not including 3)

# List methods (mutating)
numbers.append(5)         # add to end
numbers.insert(0, 0)      # insert at index 0
numbers.remove(3)        # remove first occurrence of 3
numbers.pop()            # remove and return last element
numbers.sort()           # sort in-place

# Tuples — immutable
colors = ("red", "green", "blue")
print(colors[1])    # "green"
# colors[0] = "yellow"  # ERROR — cannot modify tuple

Python

numbers = [1, 2, 3]
numbers.append(4)
print(numbers)  # [1, 2, 3, 4]

Java

ArrayList<Integer> numbers =
    new ArrayList<>();
numbers.add(1);
numbers.add(2);
// Indexing: numbers.get(0)

Java uses ArrayList<T> for dynamic lists (similar to Python lists). Methods are similar: .add(), .get(), .remove(). Java also has fixed-size int[] arrays.

🌉
Java parallel: Java's ArrayList is like Python's list. Methods: .add() (append), .get(index) (access by index), .remove(), .set(index, value) (update). Java also has arrays int[] which are fixed-size and faster. Java has no direct equivalent to tuples, but sometimes people use final to indicate immutability.
✎ Quick check
What is the key difference between a list and a tuple? When would you use a tuple?
Lists are mutable — you can change, add, or remove elements. Tuples are immutable — once created, they cannot change. Use tuples when you want to ensure data doesn't get accidentally modified, or as dictionary keys (since tuples are hashable but lists are not). Use lists when you need flexibility.

Dictionaries

Lesson 6: Key-value pairs, access, iteration
›

Dictionaries are unordered collections of key-value pairs. Access values by key. Keys are typically strings or numbers.

# Creating dictionaries
person = {
    "name": "Alice",
    "age": 25,
    "city": "Sydney"
}

# Accessing values
print(person["name"])      # "Alice"
print(person.get("age"))  # 25 (safer, returns None if missing)

# Modifying
person["age"] = 26         # update
person["email"] = "alice@example.com"  # add new key
del person["city"]        # remove key

# Iterating
for key in person:
    print(key, "→", person[key])

for key, value in person.items():
    print(key, "→", value)

# Check membership
print("name" in person)  # True

Python

d = {"x": 1, "y": 2}
d["z"] = 3
print(d.get("w"))  # None

Java

HashMap<String, Integer> d =
    new HashMap<>();
d.put("x", 1);
d.get("w");  // null (if missing)

Java's HashMap<K,V> is the equivalent. Methods: .put(key, value) (add/update), .get(key), .remove(key), .containsKey(key).

🌉
Java parallel: Java's HashMap requires you to specify key and value types: HashMap<String, Integer>. Key methods: .put(k, v) (insert/update), .get(k), .remove(k), .keySet(), .values(), .entrySet(). Iteration uses for (String key : map.keySet()) or for (Map.Entry<K,V> e : map.entrySet()).
✎ Quick check
What is the difference between d[key] and d.get(key) in Python?
d[key] raises a KeyError if the key doesn't exist. d.get(key) returns None if the key is missing (or a default value if you specify one: d.get(key, default)). Use .get() when you're unsure if the key exists.

Functions

Lesson 7: Defining, parameters, return values, scope
›

Functions are reusable blocks of code. They take parameters (inputs) and return values (outputs). They help organize code.

# Defining a function
def greet(name):
    return f"Hello, {name}!"

# Calling it
message = greet("Alice")  # "Hello, Alice!"

# Multiple parameters and return values
def add(a, b):
    return a + b

def divide_with_remainder(a, b):
    return a // b, a % b  # return tuple

quotient, remainder = divide_with_remainder(17, 5)

# Default parameters
def power(base, exp=2):
    return base ** exp

print(power(3))      # 9 (uses default exp=2)
print(power(3, 3))    # 27 (exp=3)

# Scope: local vs global
x = 10  # global variable

def func():
    x = 5   # local variable (different from global x)
    print(x)  # 5

func()
print(x)  # 10 (global x unchanged)

Python

def add(a, b):
    return a + b

result = add(3, 5)

Java

public static int add(int a, int b) {
    return a + b;
}

int result = add(3, 5);

Java functions are called methods (always part of a class). You must declare parameter types and return type. public static means the method is accessible from anywhere and doesn't need an object instance.

🌉
Java parallel: Java syntax: public static ReturnType methodName(ParamType param1, ParamType param2) { return ...; }. All types are required upfront. Java also has method overloading — you can have multiple methods with the same name but different parameter types. Scope in Java: variables declared inside a method are local to that method (same as Python).
✎ Quick check
What happens if you modify a local variable inside a function — does it affect the global variable with the same name?
No. When you assign to a variable inside a function, Python creates a local variable by default. This shadows (hides) any global variable with the same name, but doesn't modify it. To modify a global variable inside a function, use the global keyword. (Though this is generally bad practice — it's better to pass values as parameters and return results.)

File I/O

Lesson 8: Reading, writing, the 'with' statement
›

Files let you persist data. Use open() to read or write. The with statement ensures files close safely.

# Reading a file
with open("data.txt", "r") as f:
    content = f.read()      # read entire file
    # or: lines = f.readlines()  # read as list of lines

# Writing to a file
with open("output.txt", "w") as f:
    f.write("Hello, world!")

# Appending (add to end)
with open("log.txt", "a") as f:
    f.write("New log entry\n")

# Iterating over lines
with open("data.txt") as f:
    for line in f:
        print(line.strip())  # remove newline

Python

with open("file.txt", "r") as f:
    data = f.read()

Java

try (Scanner f =
    new Scanner(new File("file.txt"))) {
    String line;
    while (f.hasNextLine()) {
        line = f.nextLine();
    }
}

Java's try-with-resources statement (Java 7+) is similar to Python's with. It automatically closes the resource when done. Reading requires Scanner or BufferedReader; writing uses PrintWriter.

🌉
Java parallel: Java uses try-with-resources: try (Resource r = new Resource(...)) { ... }. For reading text: Scanner (simple) or BufferedReader (faster). For writing: PrintWriter. Both handle closing automatically, preventing file leaks.
✎ Quick check
Why is the with statement important for file handling?
The with statement automatically closes the file when you're done, even if an error occurs. Without it, you'd have to manually call f.close(), which is easy to forget. An unclosed file can leak resources. Always use with open(...) as f: instead of manually managing file closure.

Error Handling

Lesson 8: try/except, common exceptions
›

try/except blocks catch errors (exceptions) so your program doesn't crash. Python has many built-in exception types.

# Basic try/except
try:
    age = int(input("Age: "))
except ValueError:
    print("That's not a number!")

# Multiple exceptions
try:
    result = 10 / x
    data = open("file.txt")
except ZeroDivisionError:
    print("Cannot divide by zero")
except FileNotFoundError:
    print("File not found")
except Exception as e:
    print(f"Unexpected error: {e}")

# else and finally
try:
    num = int(input("Enter a number: "))
except ValueError:
    print("Invalid input")
else:
    print(f"You entered: {num}")  # runs only if no error
finally:
    print("Done")  # always runs

Python

try:
    x = int("abc")
except ValueError:
    print("Error")

Java

try {
    int x = Integer.parseInt("abc");
} catch (NumberFormatException e) {
    System.out.println("Error");
}

Java also uses try/catch. Key difference: Java distinguishes checked exceptions (you must catch them) vs unchecked exceptions (you don't have to). Python doesn't enforce this — it's your choice.

🌉
Java parallel: Java's try/catch/finally works like Python's try/except/finally. But Java is stricter: certain exceptions (like IOException) are checked — you must catch them or declare them with throws. Unchecked exceptions (like NumberFormatException) don't require this. Java also has try-with-resources for automatic resource closure.
✎ Quick check
What is the difference between except ValueError and except Exception?
except ValueError catches only ValueError exceptions (and subclasses). except Exception catches any exception except BaseException (which includes things like KeyboardInterrupt). Be specific: catching everything with except: hides bugs. Use specific exception types when possible.

Foundations in one sentence

You mastered variables, types, control flow (if/while/for), data structures (lists/dicts), functions, file handling, and error handling — all the essential building blocks for any programming language. These same concepts exist in Java with different syntax, but the logic is identical. If you understand these well, you're ready for advanced programming.

✎ Quick check
Pick one foundation concept and explain how it changes when you move from Python to Java. What stays the same? What's different?
Example: Lists. Same: ordered sequences, indexing, iteration, methods for add/remove. Different: Python lists are dynamic and can mix types; Java's ArrayList requires a type parameter (ArrayList<Integer>) and is equally dynamic but type-safe. Syntax: Python uses .append(), Java uses .add(). Indexing: Python list[0], Java list.get(0). The concept of "a mutable, ordered collection" is identical — only the syntax and type enforcement differ.
📝 Foundations Self-Test
10 questions covering Lessons 1–8 with a focus on Python↔Java bridges. Try to answer before revealing.
Q1Remembering

What is the key difference between Python's dynamic typing and Java's static typing?

Python: a variable can hold any type at any time, and the type is determined at runtime. Java: every variable has a fixed type declared upfront, and the compiler enforces type consistency. Java catches type mismatches at compile time; Python only finds them at runtime.
Q2Understanding

Why does input() always return a string in Python, even if the user types a number?

input() reads from the keyboard, which is inherently text-based. It returns everything the user types as a string, exactly as typed. If you need a number, you must explicitly convert it using int() or float().
Q3Applying

Write a for loop that prints the numbers 0 through 4.

for i in range(5):
    print(i)
range(5) generates 0, 1, 2, 3, 4 (stops before 5).
Q4Understanding

What is the difference between a list and a tuple in Python?

Lists are mutable — you can change, add, or remove elements after creation. Tuples are immutable — once created, they cannot be changed. Use tuples when you want to guarantee data integrity; use lists when you need flexibility.
Q5Applying

Write a dictionary that stores student names as keys and grades as values. Then access the grade for "Bob".

grades = {"Alice": 90, "Bob": 85, "Charlie": 95}
print(grades["Bob"]) # 85
Or safer: print(grades.get("Bob", "Not found"))
Q6Understanding

What does the with statement do when used with file operations?

The with statement automatically closes the file when the block exits, even if an error occurs. This prevents file leaks and ensures resources are cleaned up properly. Always use with open() instead of manually managing file closure.
Q7Applying (Bridge)

How would you write the Python comparison x > 5 and y < 10 in Java?

(x > 5) && (y < 10)
Replace and with &&. Java requires parentheses around the entire condition in if statements, but the operators are similar: && (and), || (or), ! (not).
Q8Understanding (Bridge)

In Java, what would be the equivalent of Python's list.append(5) using an ArrayList?

ArrayList<Integer> list = new ArrayList<>();
list.add(5);
Java's .add() is the equivalent of Python's .append(). For dictionaries: Python d["key"] = value becomes Java map.put("key", value).
Q9Analysing

If a variable called name is defined globally in Python, and you create a local variable also called name inside a function, what happens to the global name?

The global name is unaffected. When you assign to a local variable with the same name, Python creates a new local variable that shadows (hides) the global one. To modify the global variable from inside a function, use the global keyword. (Though this is generally poor design — prefer passing values as parameters.)
Q10Analysing (Bridge)

You have a try/except block in Python. What is the Java equivalent, and are there any important differences?

Python: try/except/finally. Java: try/catch/finally. Same logic, different keyword (except vs catch). Key difference: Java enforces checked exceptions — certain exceptions (like IOException) MUST be caught or declared with throws. Python has no such enforcement; it's optional.
