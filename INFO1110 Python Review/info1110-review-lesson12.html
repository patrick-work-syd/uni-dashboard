What Are Modules?

Reusable code packages — the foundation of software organization
›

A module is simply a file (or package of files) containing Python code — functions, classes, variables, and constants — that you can reuse in other programs.

Real-World Analogy

Think of modules like libraries in real life. A library contains books (functions) that you can borrow (import) without rewriting them yourself. You don't need to own the book; you borrow it when you need it. The Python Standard Library is like a massive library with thousands of pre-written books ready to use.

When you write import sys or import csv, you're borrowing code from a module file. The module sys contains functions and variables about the Python system (command-line arguments, exit codes, platform info). The module csv has functions for reading and writing CSV files.

✅
Modules are how Python avoids "reinventing the wheel" — you write a function once, put it in a module, and use it everywhere.
✎ Quick check
Name a task you'd need to do often in a program (e.g., read a file, do math, manipulate dates). What module would you import?
Examples: File I/O: import os (file paths), import pathlib (modern file handling). Math: import math (sqrt, sin, cos), import statistics (mean, median). Dates/Times: import datetime. Regular expressions: import re. The pattern is: identify the task, then search for a module that solves it (Google "python module for X").

Import Patterns: Three Ways to Import

import module · from module import func · import module as alias
›

Python gives you three ways to import code. Each is best for different situations.

Pattern 1: Full Module Import

import math
result = math.sqrt(16)  # Use module.function()

Pattern 2: Import Specific Function

from math import sqrt
result = sqrt(16)  # Use function() directly

Pattern 3: Import with Alias

import pandas as pd
df = pd.read_csv('data.csv')  # Use shortened name

When to use each:

  • Pattern 1 (full import): Safe, clear, avoids name collisions. Use by default.
  • Pattern 2 (from ... import): When you use one function repeatedly and want less typing. Risky if multiple modules have the same function name.
  • Pattern 3 (as alias): When the module name is long (e.g., numpy as np, pandas as pd). Industry standard for many libraries.
🌉
Java parallel: Java also uses import, but works at the class level: import java.util.ArrayList; or import java.util.*; (wildcard). Java doesn't have aliases — you use new ArrayList<>() directly. But the philosophy is the same: reuse pre-written code by importing it.
✎ Quick check
If you want to use the sqrt function from the math module, and you're only using it once or twice, which import pattern is best and why?
Pattern 1 is safest: import math then math.sqrt(16). It's clearer where the function comes from (it says "math.sqrt", not just "sqrt"). If another module also has a sqrt, there's no confusion. Pattern 2 is fine for quick scripts, but Pattern 1 is the professional practice.

The Python Standard Library

What's available out of the box (and what you'll encounter in INFO1113)
›

Python comes with a massive "standard library" — thousands of pre-written modules. You don't need to install anything; they're part of Python. Here are the ones you'll see most:

  • os — Operating system: list files, change directories, get environment variables.
  • sys — System info: command-line arguments (sys.argv), exit the program (sys.exit()).
  • csv — Read and write CSV files (spreadsheet-like data).
  • json — Read and write JSON data (common web format).
  • re — Regular expressions: search and manipulate text with patterns.
  • datetime — Create and manipulate dates and times.
  • math — Math functions: sqrt, sin, cos, pi, etc.
  • statistics — Basic stats: mean, median, standard deviation.
  • random — Generate random numbers and shuffle lists.

Beyond the standard library, the PyPI ecosystem (Python Package Index) has 500,000+ third-party packages for everything from web scraping to machine learning. You install them with pip install package-name. NumPy (which we'll cover) is a famous third-party package.

💡
Pro tip: Before writing code to solve a problem, search "python module for X". Odds are someone's already written it.
✎ Quick check
You need to read a JSON file and extract data from it. What module would you import?
import json. Then use json.load(file) to read a JSON file or json.loads(string) to parse a JSON string.

Creating Your Own Module

Save functions in a file, import them elsewhere. The __name__ == '__main__' guard
›

You can create your own modules. Just write Python code in a .py file and import it.

Step 1: Create a module file (math_helpers.py)

def add(a, b):
    return a + b

def multiply(a, b):
    return a * b

PI = 3.14159

Step 2: Import and use it (main.py in same folder)

import math_helpers

result = math_helpers.add(3, 5)
area = math_helpers.PI * 10 ** 2

The __name__ == '__main__' guard: When you run a Python file directly, Python sets a special variable __name__ to '__main__'. When you import the file as a module, __name__ is the module's name. This guard lets you have test code that only runs when you run the file directly, not when you import it.

# math_helpers.py
def add(a, b):
    return a + b

if __name__ == '__main__':
    # This only runs if you do: python math_helpers.py
    # NOT if you do: import math_helpers from main.py
    print(add(2, 3))  # prints 5
🌉
Java parallel: In Java, every .java file IS a class. There's no separate concept of "modules" — each file is one class with a main() method that only runs if you run that class directly. So Java's class-per-file approach is similar to Python's module pattern, just more rigid and mandatory.
✎ Quick check
You create a file utils.py with a function greet(name). In another file, how would you import and use it?
import utils then utils.greet("Will"). Or use from utils import greet then greet("Will").

NumPy: Fast Numerical Computation

Why Python lists are slow for math. NumPy arrays are contiguous memory.
›

NumPy (Numerical Python) is a third-party library for scientific computing. It's one of Python's superpowers — it makes math operations on large arrays lightning fast.

Why is NumPy fast? A Python list is a collection of pointers to objects scattered around memory. Adding two lists element-by-element is slow because Python has to jump around in memory and unbox each object. A NumPy array stores raw numbers in contiguous memory — like a C-style array. This means the CPU can process it at lightning speed.

Slow: Python lists

nums = [1, 2, 3, 4, 5]
result = [x * 2 for x in nums]
# Python loops through each item

Fast: NumPy arrays

import numpy as np
nums = np.array([1, 2, 3, 4, 5])
result = nums * 2
# Vectorized: CPU processes all at once

NumPy operations are vectorized — you write code that operates on entire arrays at once, and NumPy runs it at C speed. For arrays with millions of elements, NumPy is 100× faster than Python loops.

Common NumPy operations:

import numpy as np

# Create arrays
arr = np.array([1, 2, 3])
zeros = np.zeros(5)  # [0. 0. 0. 0. 0.]
ones = np.ones(3)   # [1. 1. 1.]
range_arr = np.arange(0, 10, 2)  # [0 2 4 6 8]

# Math operations (vectorized)
arr * 2     # [2 4 6]
arr + 10    # [11 12 13]
arr ** 2    # [1 4 9]

# Aggregation
arr.sum()     # 6
arr.mean()    # 2.0
arr.max()     # 3
arr.std()     # standard deviation
🌉
Java parallel: Java has native arrays (e.g., int[] arr = {1, 2, 3}) that ARE contiguous memory — similar to NumPy's approach. Java also has ArrayList for dynamic sizing. But Java doesn't have vectorized math like NumPy. For scientific computing in Java, you'd either use a loop or import a library.
✎ Quick check
Create a NumPy array with the numbers 10, 20, 30. Then multiply each element by 5 and print the result.
import numpy as np
arr = np.array([10, 20, 30])
result = arr * 5
print(result) # [50 100 150]

The If Expression (Ternary Operator)

Inline conditionals for concise code
›

Python has a compact way to write an if-else statement in a single expression: the if expression (also called ternary operator).

Verbose: if-else statement

if score >= 50:
    result = "Pass"
else:
    result = "Fail"

Concise: if expression

result = "Pass" if score >= 50 else "Fail"

Format: value_if_true if condition else value_if_false

The order is a bit backwards compared to English ("Pass if score >= 50 else Fail"), so read it carefully. It's useful for one-liners, but don't overuse it — if the condition is complex, use a regular if-else statement for readability.

Examples:

age = 25
status = "Adult" if age >= 18 else "Minor"
# status = "Adult"

sign = "positive" if num > 0 else "non-positive"

max_val = a if a > b else b
# Could also use: max(a, b)
🌉
Java parallel: Java's ternary operator: condition ? trueVal : falseVal. Same idea, different syntax. Example: String status = (age >= 18) ? "Adult" : "Minor";
✎ Quick check
Write a single line of Python that assigns "Even" to a variable if num is divisible by 2, otherwise "Odd".
parity = "Even" if num % 2 == 0 else "Odd"

Set Comprehensions

Creating sets with a single expression
›

A set comprehension is like a list comprehension but creates a set (unique values) instead of a list. Use curly braces { } instead of square brackets [ ].

List comprehension (list)

numbers = [1, 2, 2, 3, 3, 3]
squares = [x ** 2 for x in numbers]
# [1, 4, 4, 9, 9, 9]

Set comprehension (set)

numbers = [1, 2, 2, 3, 3, 3]
squares = {x ** 2 for x in numbers}
# {1, 4, 9}  ← unique only

Format: {expression for item in iterable if condition}

The difference: a list keeps duplicates and maintains order; a set removes duplicates automatically. Use set comprehensions when you want only unique values.

Examples:

# Get unique vowels from a word
word = "programming"
vowels = {char for char in word if char in "aeiou"}
# {'a', 'o', 'i'}

# Get unique lengths of words
words = ["apple", "cat", "dog", "bird"]
lengths = {len(w) for w in words}
# {3, 4, 5}
✎ Quick check
Write a set comprehension to get unique digits from the number 112233 (convert to string first). The result should be {'1', '2', '3'}.
digits = {char for char in "112233"} or {char for char in str(112233)}. Result: {'1', '2', '3'}

Dictionary Comprehensions

Creating dictionaries from data in one line
›

A dictionary comprehension builds a dictionary using a single expression. It's the dictionary equivalent of a list comprehension.

Without comprehension (verbose)

products = ["apple", "banana", "orange"]
prices = [1.50, 0.75, 2.00]

product_dict = {}
for i in range(len(products)):
    product_dict[products[i]] = prices[i]

With comprehension (concise)

products = ["apple", "banana", "orange"]
prices = [1.50, 0.75, 2.00]

product_dict = {p: pr for p, pr in zip(products, prices)}

Format: {key: value for item in iterable if condition}

The key thing is the key: value syntax. You specify what the key and value should be for each iteration.

More examples:

# Build a dict of word → length
words = ["cat", "dog", "elephant"]
word_lengths = {w: len(w) for w in words}
# {'cat': 3, 'dog': 3, 'elephant': 8}

# Build a dict of number → square
numbers = [1, 2, 3, 4, 5]
squares = {n: n ** 2 for n in numbers}
# {1: 1, 2: 4, 3: 9, 4: 16, 5: 25}

# Filter with condition
even_squares = {n: n ** 2 for n in numbers if n % 2 == 0}
# {2: 4, 4: 16}
🌉
Java parallel: Java has no dict comprehensions. You'd use a loop with a HashMap: HashMap<String, Integer> map = new HashMap<>(); then loop and put values in. Or in modern Java 8+, use Streams with Collectors.toMap() (which is closer to comprehension style, but more verbose).
✎ Quick check
Write a dictionary comprehension that maps numbers 1-5 to their word equivalents: 1→"one", 2→"two", etc.
words = ["one", "two", "three", "four", "five"]
num_dict = {i: words[i-1] for i in range(1, 6)}
# {1: 'one', 2: 'two', 3: 'three', 4: 'four', 5: 'five'}

Lesson 12 in one sentence

Modules let you organize and reuse code; imports bring that code into your program; NumPy gives Python fast numerical computation; and comprehensions are syntactic sugar for building lists, sets, and dicts — all of which have equivalents in Java, though Java's approach is more verbose and type-heavy.

✎ Quick check
Name two key differences between how Python and Java organize and reuse code, and one way they're similar.
Differences: (1) Python has modules (files); Java has classes and packages (more rigid, one class per file by convention). (2) Python doesn't require type declarations for imports; Java does. Similar: Both use import statements to bring external code into your program. The philosophy of code reuse is identical.

Q1Remembering

What is a module?

A file (or package of files) containing reusable Python code — functions, classes, variables. You import a module to use its code without rewriting it.
Q2Understanding

What's the difference between import csv and from csv import reader?

import csv brings the entire module in; you use csv.reader(). from csv import reader brings just the reader function; you use reader() directly. The second is shorter but can cause name collisions if another module has a function named reader.
Q3Remembering

Name three modules from the Python Standard Library and what they're used for.

Examples: os (file operations), datetime (dates/times), math (math functions), json (JSON parsing), re (regular expressions), csv (CSV files), random (random numbers).
Q4Understanding

What is the if __name__ == '__main__': guard used for?

It lets you write test or demo code that only runs when you run the file directly (not when the file is imported as a module by another program). When a file is imported, __name__ is the module name, not '__main__'.
Q5Applying

Why is NumPy fast for numerical operations compared to Python lists?

Python lists are arrays of pointers scattered in memory. NumPy arrays store raw numbers in contiguous memory (like C arrays). This lets the CPU process them without jumping around, and without the overhead of Python's object system. The result: NumPy is 100× faster for large arrays.

Q9Analysing (Bridge)

In Java, how would you import a specific class from a package, and how does this compare to Python's from module import function?

Java: import java.util.ArrayList; (specific class) vs import java.util.*; (wildcard). Python: from collections import Counter is similar to Java's specific import. Both let you use the class/function directly without the package prefix. The main difference: Java imports are at compile time; Python imports are at runtime and can be conditional.
Q10Analysing (Bridge)

Name one thing Python's comprehensions make easy that would require a loop in Java.

Creating a new collection (list, set, dict) from existing data with filtering/transformation. Example: squares = [x**2 for x in nums if x > 0] in Python is one line. In Java, you'd need a loop: List<Integer> squares = new ArrayList<>(); for (int x : nums) { if (x > 0) squares.add(x * x); } (or use Streams, which is more concise but still more verbose than Python).
