What Is a Class? The Mental Model

Blueprint vs. instance — the core idea before any code
›

Before writing code, you need to understand what a class actually is. Everything else builds on this one idea.

Real-World Analogy

A class is like an architectural blueprint for a house. The blueprint defines the structure (3 bedrooms, 2 bathrooms, a kitchen) but it's not a house you can live in. An object (or instance) is an actual house built from that blueprint. You can build many houses from the same blueprint — each one is independent but has the same structure.

In programming terms:

  • Class = a template that defines what data (attributes) and actions (methods) an object will have
  • Object / Instance = a specific copy created from that template, with its own data
  • Instantiation = the act of creating an object from a class

The key insight: one class, many objects. Each object has the same structure but its own independent data. Changing Bob's weight doesn't affect Alice.

✎ Quick check
If you create three Person objects from the same class, and you change one person's name, what happens to the other two?
Nothing. Each object is an independent instance with its own copy of the data (attributes). Changing one object's attribute does not affect any other object. They share the same structure (defined by the class) but have separate data.

Defining a Class in Python

class, __init__, self, and instance variables
›

In Python, you define a class with the class keyword. The __init__ method is the constructor — it runs automatically when you create a new object and sets up the initial data.

class Person:
    def __init__(self, name, weight, height):
        self.name = name        # instance variable
        self.weight = weight    # instance variable
        self.height = height    # instance variable

    def greet(self):
        return f"Hi, I'm {self.name}"

# Create two objects (instances)
bob = Person("Bob", 82, 178)
alice = Person("Alice", 65, 165)

print(bob.greet())    # Hi, I'm Bob
print(alice.name)     # Alice

Key pieces explained

  • self = refers to the specific object being created or used. Every method's first parameter is self — Python passes the object automatically.
  • self.name = name = stores the parameter name as an attribute inside the object. Without self., it's just a local variable that disappears.
  • __init__ = the initialiser (constructor). It doesn't create the object — Python does that. It configures the fresh object with its starting data.
⚠️
Common mistake: forgetting self in method parameters. If you write def greet(): instead of def greet(self):, Python will throw TypeError: greet() takes 0 positional arguments but 1 was given because it tries to pass the object as the first argument.
🌉

Java Bridge

In Java, self becomes this — and it's implicit (you don't write it as a parameter). The constructor is a method with the same name as the class, not __init__.

Python

class Person:
    def __init__(self, name):
        self.name = name

    def greet(self):
        return f"Hi, {self.name}"

p = Person("Bob")

Java

public class Person {
    String name;

    public Person(String name) {
        this.name = name;
    }

    public String greet() {
        return "Hi, " + name;
    }
}
Person p = new Person("Bob");

✎ Quick check
What does self refer to in self.name = name? Why is it necessary?
self refers to the specific object being constructed. self.name stores the value as an attribute of the object, so it persists after __init__ finishes. Without self., name = name would just be a local variable that vanishes when the method ends — the object would have no data.

Instance Variables vs. Class Variables

Data that belongs to ONE object vs. data shared by ALL objects
›

There are two types of data a class can hold. Understanding the difference is critical for Java, where this distinction is enforced with the static keyword.

class Dog:
    species = "canine"           # CLASS variable — shared by ALL dogs

    def __init__(self, name):
        self.name = name          # INSTANCE variable — unique to each dog

rex = Dog("Rex")
milo = Dog("Milo")

print(rex.name)       # Rex — instance variable, unique to rex
print(milo.name)      # Milo — instance variable, unique to milo
print(rex.species)    # canine — class variable, shared
print(Dog.species)    # canine — can access through the class itself
🌉

Java Bridge

In Java, class variables use the static keyword. Instance variables don't. This is one of the first things you'll see in INFO1113.

Python

class Dog:
    species = "canine"  # class var

    def __init__(self, name):
        self.name = name  # instance

Java

public class Dog {
    static String species = "canine";
    String name;  // instance

    public Dog(String name) {
        this.name = name;
    }
}
💡
Rule of thumb: if the data is the same for every object of that type, it's a class/static variable. If the data is different per object (like a name or score), it's an instance variable.
✎ Quick check
If you write rex.species = "feline", does milo.species also change?
No! This is a Python subtlety. Assigning rex.species = "feline" creates a new instance variable on rex that shadows the class variable. milo.species still reads from the class variable "canine". To change it for all dogs, use Dog.species = "feline". In Java, you'd use Dog.species = "feline" (via the class name) to change the static variable.

Methods: Actions Your Objects Can Perform

Instance methods, reading from files, and building real programs
›

Methods are functions that live inside a class and operate on the object's data. The professor's lecture walks through a full example: a Person class that reads data from a CSV file and stores each person as an object.

class Person:
    def __init__(self, name, weight, height):
        self.name = name
        self.weight = weight
        self.height = height

    def bmi(self):
        # BMI = weight(kg) / height(m)²
        height_m = self.height / 100
        return self.weight / (height_m ** 2)

    def describe(self):
        return f"{self.name}: BMI = {self.bmi():.1f}"

bob = Person("Bob", 82, 178)
print(bob.describe())  # Bob: BMI = 25.9

Notice how describe() calls self.bmi() — methods can call other methods on the same object. This is a key pattern: objects know how to do things with their own data.

🌉
Java parallel: In INFO1113, every piece of code lives inside a class. The bmi() method would look like public double bmi() { return weight / Math.pow(height / 100.0, 2); } — same logic, but with types declared and Math.pow instead of **.
✎ Quick check
In the describe() method, why do we write self.bmi() and not just bmi()?
bmi() is a method that belongs to the object, not a standalone function. self.bmi() tells Python to call the bmi method on this particular object, using this object's weight and height. Without self., Python would look for a regular function named bmi and throw a NameError.

Special Methods (Magic Methods)

__repr__, __eq__, __str__, __lt__ — customising object behaviour
›

Python lets you define special methods (also called dunder/magic methods) that control how objects behave with built-in operations like printing, comparing, and sorting.

class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __repr__(self):
        return f"Point({self.x}, {self.y})"

    def __eq__(self, other):
        return isinstance(other, Point) and self.x == other.x and self.y == other.y

    def __lt__(self, other):
        return (self.x ** 2 + self.y ** 2) < (other.x ** 2 + other.y ** 2)

a = Point(1, 2)
b = Point(1, 2)
print(a)         # Point(1, 2)  — __repr__ controls this
print(a == b)    # True         — __eq__ controls this
print(a < Point(3, 4))  # True — __lt__ controls this

⚠️
Java gotcha: In Java, == compares references (memory addresses), not values. You must use .equals() to compare objects by value. This trips up everyone coming from Python where == calls __eq__.
✎ Quick check
Without defining __eq__, what does Point(1,2) == Point(1,2) return? Why?
False. Without a custom __eq__, Python uses the default behaviour inherited from object, which compares identity (memory addresses) — the same as Java's ==. Two separate Point objects at different memory locations are not identical, even if they have the same x and y values.

Private Variables & Encapsulation

Controlling access to object data — Python conventions vs. Java enforcement
›

Encapsulation means hiding an object's internal data and only exposing it through controlled methods. This prevents code outside the class from breaking the object's internal state.

class BankAccount:
    def __init__(self, owner, balance):
        self.owner = owner
        self.__balance = balance    # __ prefix = "private"

    def deposit(self, amount):
        if amount > 0:
            self.__balance += amount

    def get_balance(self):
        return self.__balance

acct = BankAccount("Bob", 1000)
print(acct.get_balance())    # 1000 — access through method
print(acct.__balance)        # AttributeError! Can't access directly

Python uses naming conventions to signal privacy:

  • _name (single underscore) = "please don't touch this from outside" — a convention, not enforced
  • __name (double underscore) = name-mangled to _ClassName__name — harder to access accidentally
🌉

Java Bridge — This Is ENFORCED in Java

Java doesn't rely on conventions. It has real access modifiers that the compiler enforces. This is one of the biggest differences.

Python (convention)

class BankAccount:
    def __init__(self, bal):
        self.__balance = bal # "private"

    def get_balance(self):
        return self.__balance

Java (enforced)

public class BankAccount {
    private double balance;

    public double getBalance() {
        return balance;
    }
}

💡
Why this matters for Java: In INFO1113, you'll be expected to make fields private and provide public getter/setter methods. This "getters and setters" pattern comes directly from encapsulation.
✎ Quick check
Why would you use private fields with public getter methods instead of just making the field public?
Control and safety. With a public field, any code can set it to an invalid value (e.g., negative balance). With a private field and a setter method, you can add validation: if (amount > 0) balance += amount;. You control how the data is changed, not just whether it can be changed.

Inheritance: Building on Existing Classes

Reuse code by creating specialised versions of a class
›

Inheritance lets you create a new class based on an existing one. The new class (child/subclass) automatically gets all the attributes and methods of the existing class (parent/superclass), and can add or modify them.

Real-World Analogy

A "Vehicle" is a general concept (has wheels, can move). A "Car" is a specific type of Vehicle (has 4 wheels, has doors). A "Truck" is another type (bigger, has a cargo bed). Car and Truck inherit from Vehicle — they share the common parts but add their own specialisations.

class Animal:
    def __init__(self, name, sound):
        self.name = name
        self.sound = sound

    def speak(self):
        return f"{self.name} says {self.sound}"

class Dog(Animal):                  # Dog inherits from Animal
    def __init__(self, name, breed):
        super().__init__(name, "Woof")  # call parent's __init__
        self.breed = breed             # add new attribute

    def fetch(self):                  # add new method
        return f"{self.name} fetches the ball!"

rex = Dog("Rex", "Labrador")
print(rex.speak())    # Rex says Woof  — inherited from Animal
print(rex.fetch())    # Rex fetches the ball!  — Dog's own method

Key mechanics

  • class Dog(Animal) — parentheses indicate Dog extends Animal
  • super().__init__(...) — calls the parent class's constructor to set up inherited attributes
  • Dog inherits speak() without rewriting it
  • Dog adds fetch() and breed — new behaviour
🌉

Java Bridge

Java uses extends instead of parentheses, and super() works the same way. This is Week 3 material in INFO1113 — you'll use it constantly.

Python

class Dog(Animal):
    def __init__(self, name, breed):
        super().__init__(name, "Woof")
        self.breed = breed

Java

public class Dog extends Animal {
    String breed;

    public Dog(String name, String breed) {
        super(name, "Woof");
        this.breed = breed;
    }
}
⚠️
Common mistake: forgetting to call super().__init__(). If you don't, the parent class's attributes (like name and sound) never get set, and you'll get AttributeError when trying to access them. Same in Java — the compiler will error if the parent constructor needs arguments and you don't call super().
✎ Quick check
Can a Dog object call speak() even though Dog doesn't define that method? Why?
Yes. Dog inherits speak() from Animal. When Python can't find a method on the Dog class, it looks up the inheritance chain to the parent class (Animal), finds speak() there, and uses it. This is the whole point of inheritance — reuse without rewriting.

Method Overriding & Polymorphism

When a child class replaces a parent's method — the foundation of flexible code
›

Sometimes a subclass needs to change how an inherited method works. This is called overriding — the child provides its own version of the method.

class Animal:
    def __init__(self, name):
        self.name = name

    def speak(self):
        return f"{self.name} makes a sound"

class Cat(Animal):
    def speak(self):                        # OVERRIDES parent's speak
        return f"{self.name} says Meow"

class Dog(Animal):
    def speak(self):                        # OVERRIDES parent's speak
        return f"{self.name} says Woof"

# Polymorphism in action:
animals = [Cat("Whiskers"), Dog("Rex"), Animal("Mystery")]
for a in animals:
    print(a.speak())
# Whiskers says Meow
# Rex says Woof
# Mystery makes a sound

Polymorphism = "many forms." The same method call (a.speak()) produces different behaviour depending on the actual type of the object. The code doesn't need to know whether a is a Cat, Dog, or generic Animal — it just calls speak() and the right version runs.

🌉
Java parallel: Overriding works identically in Java. The key difference: Java uses the @Override annotation (optional but strongly recommended) to tell the compiler "I intend to override this method." If you misspell the method name, the compiler catches it. Python has no such safety net.
✎ Quick check
If you have a list of Animal objects and call speak() on each, how does Python know which version of speak() to run?
Python checks the actual type of the object at runtime, not the variable type. If the object is a Cat, Python finds Cat's speak() first (since it's closer in the method resolution order). If the object is a plain Animal, Python uses Animal's speak(). This is dynamic dispatch — the decision happens at runtime based on the real object.

isinstance() and Type Checking

Checking what type an object is — and why it matters
›

With inheritance, an object can be multiple types at once. A Dog is both a Dog and an Animal. Python's isinstance() checks this correctly.

rex = Dog("Rex", "Labrador")

print(isinstance(rex, Dog))     # True  — rex is a Dog
print(isinstance(rex, Animal))  # True  — rex is ALSO an Animal
print(isinstance(rex, Cat))     # False — rex is not a Cat
print(type(rex) == Dog)         # True  — exact type is Dog
print(type(rex) == Animal)      # False — type() checks EXACT type only

isinstance() respects inheritance — it checks the entire chain. type() only checks the exact type. Always prefer isinstance() for type checking, especially in __eq__ methods.

🌉
Java parallel: Java uses instanceof (one word, no parentheses): if (rex instanceof Animal). Same concept, slightly different syntax. You'll use this in INFO1113 when checking object types before casting.
✎ Quick check
Why does isinstance(rex, Animal) return True even though rex was created as a Dog?
Because Dog inherits from Animal. Every Dog is-a Animal. isinstance() walks up the inheritance chain: Is rex a Dog? Yes. Is rex an Animal? Dog extends Animal, so yes. This "is-a" relationship is the whole point of inheritance — subclass objects are valid wherever the parent type is expected.

📝 Lesson 11 Self-Test
12 questions covering classes, inheritance, and the Python→Java bridge. Try to answer before revealing.
Q1Remembering

What is the difference between a class and an object (instance)?

A class is a blueprint/template that defines attributes and methods. An object (instance) is a specific copy created from that class, with its own data. You can create many objects from one class.
Q2Understanding

Why does every Python method need self as the first parameter? What does self actually refer to?

self is a reference to the specific object the method is being called on. It's how the method accesses the object's data. When you call bob.greet(), Python passes bob as the self argument automatically. Without self, the method couldn't access or modify any of the object's attributes.
Q3Remembering

What's the difference between self.name (instance variable) and a variable defined at class level (class variable)?

An instance variable (self.name) belongs to one specific object — each object has its own copy. A class variable (defined outside __init__) is shared by all instances. In Java, class variables use the static keyword.
Q4Applying

Write a Python class Rectangle with width and height, and an area() method that returns width × height.

class Rectangle:
  def __init__(self, width, height):
    self.width = width
    self.height = height
  def area(self):
    return self.width * self.height
Q5Understanding

Explain what super().__init__(name, "Woof") does inside a Dog class that inherits from Animal.

It calls the parent class (Animal)'s constructor, passing name and "Woof" as arguments. This sets up the inherited attributes (self.name and self.sound) that Animal's __init__ defines. Without this call, the Dog object would be missing those attributes.
Q6Applying

Create a Student class that inherits from Person. Student adds a student_id attribute. Show the __init__ method.

class Student(Person):
  def __init__(self, name, student_id):
    super().__init__(name)
    self.student_id = student_id
Q7Analysing

Why does isinstance(rex, Animal) return True when rex is a Dog? What principle does this demonstrate?

Because Dog inherits from Animal, every Dog is-a Animal. isinstance() walks up the inheritance chain. This demonstrates the Liskov Substitution Principle — anywhere an Animal is expected, a Dog should work. In Java, this is called substitutability.
Q8Analysing

What happens when you call speak() on a Cat object where Cat overrides Animal's speak()? Which version runs and why?

Cat's version runs. Python uses dynamic dispatch — it checks the actual object's type at runtime. Since the object is a Cat, Python finds Cat's speak() method first (before looking at the parent). This is polymorphism: the same method call produces different behaviour based on the actual object type.
Q9Applying (Bridge)

Translate this Python class to Java: class Dog: species = "canine"; def __init__(self, name): self.name = name

  String name;

Key changes: static for class variable, type declarations, this instead of self, constructor named Dog not __init__.
Q10Understanding (Bridge)

In Python, == calls __eq__. What does == do in Java? How do you compare object values in Java?

In Java, == compares references (memory addresses), not values. Two objects with identical data still return false with == if they're at different memory locations. To compare values, use .equals() — which is Java's equivalent of Python's __eq__.
Q11Understanding (Bridge)

Python's self.__balance uses naming convention for privacy. How does Java enforce privacy differently?

Java uses access modifiers enforced by the compiler: private, protected, public. A private field literally cannot be accessed outside the class — the compiler rejects the code. Python's double underscore is just name-mangling (you can still access it as _ClassName__balance). Java's enforcement is absolute.
Q12Analysing (Bridge)

Name two advantages Java's strict type system and access modifiers give you over Python's flexible approach.

1. Compile-time error detection: The compiler catches type mismatches and illegal access before the code runs, so bugs are found earlier. 2. Self-documenting code: Type declarations and access modifiers tell you exactly what type each variable holds and who can access it, making large codebases easier to understand. The trade-off: more verbose code.
