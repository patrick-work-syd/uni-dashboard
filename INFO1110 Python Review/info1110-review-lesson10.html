Function Arguments, Parameters & Sets

10 steps · how Python handles function flexibility · and a new data structure

This lesson covers Python's powerful function flexibility — default values, variable arguments (*args and **kwargs), and keyword arguments. These are things Java doesn't have; Java uses strict parameter lists and method overloading instead. We'll also explore sets — a data structure that Java has too (as HashSet). Understanding the Python versions will make Java's approach make sense.

Arguments vs Parameters — The Terminology

Why the distinction matters, even though people use them interchangeably
›

Most people use "arguments" and "parameters" to mean the same thing. But they're technically different, and understanding the distinction makes function signatures clearer.

Mental Model

Parameters are the placeholders defined in the function signature. Arguments are the actual values you pass when you call the function.

In this example:

def greet(name, greeting="Hello"):  # 'name' and 'greeting' are PARAMETERS
    return f"{greeting}, {name}!"

greet("Alice", "Hi")           # "Alice" and "Hi" are ARGUMENTS
  • Parameters (definition): name, greeting
  • Arguments (function call): "Alice", "Hi"
✅
In practice, almost everyone says "arguments" and "parameters" interchangeably. But during this lesson, we'll use the precise terms to be clear about function design.
✎ Quick check
In the function def add(a, b): return a + b, which are the parameters and which are the arguments when you call add(3, 5)?
Parameters: a and b (defined in the function signature). Arguments: 3 and 5 (the values you pass when you call the function).

Positional Arguments — The Basics

Standard left-to-right matching
›

Positional arguments are the simplest and most common. You pass values in the order they appear in the function definition, and Python matches them left-to-right.

def subtract(a, b):
    return a - b

subtract(10, 3)     # a=10, b=3 → returns 7
subtract(3, 10)     # a=3, b=10 → returns -7 (order matters!)
⚠️
Order matters! In positional arguments, the position determines which parameter gets which value. Swap the arguments and you get a different result.
🌉

Java parallel

Java also supports positional arguments only. There are no keyword arguments or default values in Java. You match arguments left-to-right, just like Python. But Java requires type declarations: int subtract(int a, int b) { return a - b; }

✎ Quick check
What would subtract("hello", "world") do? What error would you get?
It would raise a TypeError because you can't subtract strings (unless the function was designed differently). The arguments are positional — Python just tries to match them left-to-right and execute the function. If the types don't work, you get an error at runtime.

Default Values & Keyword Arguments

Parameters with defaults, and calling by name instead of position
›

Python lets you define default values for parameters. When you call the function, you can either pass a value (override the default) or omit it (use the default). You can also pass arguments by name (keyword argument).

def greet(name, greeting="Hello", punctuation="!"):
    return f"{greeting}, {name}{punctuation}"

greet("Alice")                                  # Uses defaults: "Hello, Alice!"
greet("Bob", "Hi")                            # Override first default: "Hi, Bob!"
greet("Carol", punctuation="?")          # Keyword arg, skip greeting: "Hello, Carol?"
greet(greeting="Hey", name="Dave")        # All keywords, any order: "Hey, Dave!"

Keyword arguments let you pass arguments by name, and they can be in any order. This is powerful because you can skip defaults you don't care about.

💡
Rule: Parameters with defaults must come after parameters without defaults. You can't have a required parameter after an optional one.
🌉

Java parallel: Method Overloading

Java has NO keyword arguments or default parameter values. Instead, Java uses method overloading — you define multiple methods with the same name but different parameter lists. public void greet(String name) { } and public void greet(String name, String greeting) { } are two different methods. You can't reorder arguments; you must pass them positionally.

✎ Quick check
What's the advantage of keyword arguments over positional arguments?
Keyword arguments let you specify which parameter you're passing a value to, without needing to pass all parameters. You can skip defaults you don't need, and you can pass them in any order. This makes code more readable: greet(punctuation="?", name="Carol") is clearer than greet("Carol", "Hello", "?").

*args — Variable Positional Arguments

Accepting any number of positional arguments as a tuple
›

Sometimes you want a function to accept any number of positional arguments. Use *args (the name "args" is just convention; it's the asterisk that matters). Python collects all extra positional arguments into a tuple.

def sum_all(*args):
    total = 0
    for num in args:
        total += num
    return total

sum_all(1)              # args=(1,) → 1
sum_all(1, 2, 3)        # args=(1,2,3) → 6
sum_all(1, 2, 3, 4, 5) # args=(1,2,3,4,5) → 15

args is a tuple inside the function, so you can iterate over it, index it, check its length — just like any tuple.

🌉

Java parallel: Varargs

Java has varargs (variable arguments): public int sumAll(int... nums). The ... is like Python's *. Inside the method, nums is an array, not a tuple. The concept is identical: accept any number of arguments of one type.

✎ Quick check
If you call sum_all() with no arguments, what does args equal?
args becomes an empty tuple: (). Your function still runs — the loop just doesn't iterate, so the total stays 0 and the function returns 0.

**kwargs — Variable Keyword Arguments

Accepting any number of keyword arguments as a dictionary
›

Use **kwargs (convention; it's the double asterisk that matters) to accept any keyword arguments. Python collects them into a dictionary.

def print_info(**kwargs):
    for key, value in kwargs.items():
        print(f"{key} = {value}")

print_info(name="Alice", age=25, city="Sydney")
# Output:
# name = Alice
# age = 25
# city = Sydney

kwargs is a dictionary inside the function, so you can access keys like kwargs["name"] or iterate with .items().

💡
You can combine *args and **kwargs in one function: def flex_func(required, *args, **kwargs):. The order matters: positional first, then *args, then **kwargs.
⚠️
Java has no direct equivalent. To pass arbitrary keyword arguments in Java, you'd use a Map<String, Object> or builder pattern. There's no syntactic sugar like Python's **kwargs.
✎ Quick check
What's the difference between *args and **kwargs?
*args collects positional arguments into a tuple. **kwargs collects keyword arguments into a dictionary. With *args, you get an ordered list. With **kwargs, you get key-value pairs so you can look up values by name.

The __name__ Guard & __main__

Running as a script vs. importing as a module
›

When Python runs a file, it sets a special variable called __name__. If you run the file directly, __name__ is "__main__". If you import it as a module, __name__ is the module name. This lets you write code that only runs when the file is executed directly.

def say_hello():
    print("Hello!")

if __name__ == "__main__":
    say_hello()  # Only runs if you execute this file directly
    # Doesn't run if another file imports this one

This guard pattern is standard in Python. It lets you write functions for import, but also include test code that runs when you test the file directly.

Why This Matters

Imagine you have a file math_utils.py with a add() function. You want to import it in another file and use add(). But you also want to run math_utils.py directly to test it. The guard lets you do both: the test code runs when you execute the file, but stays quiet when it's imported.

🌉

Java parallel: main() method

Java requires every program to have a public static void main(String[] args) method. This is where execution starts. In a way, Java forces every file to have explicit entry points. There's no "import mode" that skips main() — if you import a Java class, you're using its methods, not executing main().

✎ Quick check
What would happen if you put say_hello() outside the if __name__ == "__main__": guard?
The function would run every time the file is executed or imported. If another file does import math_utils, it would trigger the hello message immediately. That's usually not what you want — you only want test code to run in test mode, not every time someone imports your module.

The `with` Statement — Context Managers

Safe resource handling with automatic cleanup
›

The with statement is Python's way of guaranteeing cleanup. When you open a file or acquire a resource, with ensures it's closed/released even if an error happens.

# Without with (bad — may leak the file)
f = open("data.txt")
data = f.read()
f.close()  # If an error happens before this, file stays open!

# With with (good — always closes)
with open("data.txt") as f:
    data = f.read()  # File auto-closes when this block ends, even on error

The as clause assigns the file object to a variable you can use inside the block. When the block ends (normally or via exception), Python calls cleanup automatically.

🌉

Java parallel: try-with-resources

Java 7+ has try-with-resources: try (Scanner s = new Scanner(file)) { // use s }. Same concept — automatic cleanup. The syntax is different, but the goal is identical: ensure resources are cleaned up.

✎ Quick check
Why is with open(...) as f: better than opening and closing manually?
If an error occurs inside the block, the manual approach might skip the close() call, leaving the file open and wasting resources. with guarantees cleanup regardless of exceptions. It's more reliable and less error-prone.

Sets — Unordered Collections of Unique Elements

A data structure that automatically removes duplicates
›

A set is like a list, but with two key differences: (1) unordered, and (2) no duplicates. Define a set with curly braces: {1, 2, 3}.

my_list = [1, 2, 2, 3, 3, 3]  # List with duplicates
my_set = {1, 2, 3}          # Set: no duplicates

print(my_set)           # {1, 2, 3} (order may vary)

# Convert list to set to remove duplicates
no_dupes = set(my_list)  # {1, 2, 3}

2 in my_set             # True (O(1) lookup, very fast)
my_set.add(4)         # {1, 2, 3, 4}
my_set.remove(2)     # {1, 3, 4}

Key advantage: Checking if a value is in a set is very fast (O(1)), whereas checking a list is slow (O(n)).

🌉

Java parallel: HashSet

Java's HashSet<T> is equivalent to Python's set. Same properties: unordered, no duplicates, fast lookup. Methods have different names: .add(), .remove(), .contains() instead of Python's methods.

✎ Quick check
You have a list of 10,000 names with many duplicates. How would a set help you find unique names?
Convert the list to a set: unique = set(name_list). The set automatically removes duplicates, so you instantly have only the unique names. And if you need to check whether a specific name is in the list later, name in unique is much faster than checking a list of 10,000 items.

Set Operations — Union, Intersection, Difference

Combining and comparing sets with operators
›

Sets support mathematical operations that make sense: combine them (union), find common elements (intersection), find differences (what's in one but not the other).

a = {1, 2, 3}
b = {3, 4, 5}

a | b         # Union: {1, 2, 3, 4, 5}
a & b         # Intersection: {3}
a - b         # Difference: {1, 2} (in a but not b)
a ^ b         # Symmetric difference: {1, 2, 4, 5} (in a or b, but not both)

These operations are powerful for tasks like: "Find users who visited both pages" (intersection), "Find all unique products from two warehouses" (union), "Find products in warehouse A that aren't in warehouse B" (difference).

💡
Set operations are fast because sets use hash tables internally. Comparing two sets is O(n), not O(n²). For large datasets, this makes a huge difference.
🌉

Java parallel: HashSet operations

Java's HashSet has methods instead of operators: .addAll() (union), .retainAll() (intersection), .removeAll() (difference). Same operations, different syntax.

✎ Quick check
What would {1, 2, 3} ^ {2, 3, 4} return? What does symmetric difference mean?
{1, 4}. Symmetric difference is "in one set or the other, but not both." So 1 is only in the first set, 4 is only in the second. But 2 and 3 are in both, so they're excluded. It's like "difference, but both directions."

Lesson 10 in one sentence

Python's flexibility — default values, keyword arguments, *args, **kwargs — lets you write functions that accept many calling patterns. Java's strictness requires explicit parameter lists and method overloading. Both have sets (unordered unique elements); both support variable-length arguments. Understanding Python's versions helps you appreciate why Java designed it differently.

✅
Ready for INFO1113. When you see a Java method with int[] nums parameter, remember: it's like Python's *args. When Java forces you to pick one method signature, remember: it's because Python's default values and keyword arguments aren't available. And when you use HashSet in Java, you'll recognize it as the same concept as Python's set.
✎ Quick check
Name two Python function features that don't exist in Java, and explain how Java achieves similar functionality.
(1) Default parameter values: Python: def greet(name, greeting="Hi"). Java: Use method overloading — multiple methods with different parameter lists. (2) Keyword arguments: Python: func(a=1, b=2, c=3) in any order. Java: No direct equivalent; you pass by position or use a builder pattern/Map for flexibility. Java forces you to decide on exact parameter lists at design time.

Q1Remembering

What's the difference between a parameter and an argument?

A parameter is the placeholder in the function definition. An argument is the actual value you pass when calling the function. Example: in def add(a, b):, a and b are parameters. In add(3, 5), 3 and 5 are arguments.
Q2Understanding

Why does order matter in positional arguments?

Python matches arguments left-to-right with parameters. If you pass subtract(3, 10), Python assigns 3 to the first parameter and 10 to the second. Swap them and the result is different. Position determines which parameter gets which value.
Q3Remembering

What's a default value, and why would you use one?

A default value is a value a parameter gets if you don't pass an argument. Example: def greet(name, greeting="Hello") — if you call greet("Alice"), greeting defaults to "Hello". Use defaults for optional parameters you don't always need to specify.
Q4Applying

Write a function describe_pet(name, animal_type="cat") and call it two ways: with both arguments, and with just the name.

def describe_pet(name, animal_type="cat"):
  return f"{name} is a {animal_type}"

describe_pet("Fluffy", "dog") # Both
describe_pet("Whiskers") # Just name, animal_type defaults to "cat"
Q5Understanding

What's the advantage of keyword arguments over positional?

Keyword arguments let you specify which parameter you're passing to, and in any order. greet(greeting="Hi", name="Bob") is clear about what each value is for, even though the order is reversed. You can skip defaults you don't need.
Q6Remembering

What does *args collect, and what data structure is it?

*args collects all extra positional arguments into a tuple. Inside the function, args is a tuple, so you can iterate over it, index it, check its length, etc.
Q7Applying

Write a function that prints all arguments passed to it using *args.

def print_all(*args):
  for arg in args:
    print(arg)
Q8Understanding

What does if __name__ == "__main__": do, and when would you use it?

This guard checks if the file is running directly (not being imported). Code inside only runs when you execute the file directly; it's skipped if another file imports this one. Use it for test code or demo code you want to run during development but not during import.
Q9Understanding

How is a set different from a list?

A list is ordered and allows duplicates. A set is unordered and automatically removes duplicates. Also, checking if an element is in a set is much faster (O(1)) than checking a list (O(n)).
Q10Applying (Bridge)

Python has default values; Java doesn't. How would you achieve the same functionality in Java?

Use method overloading — write multiple methods with the same name but different parameter lists. Example: void greet(String name) { greet(name, "Hello"); } and void greet(String name, String greeting) { }. The first method calls the second with a default value.
Q11Applying (Bridge)

In Java, you write void sum(int... nums). What's the Python equivalent?

def sum(*nums):. Both accept any number of positional arguments of one type. Java uses ... (varargs), Python uses *. Inside the method/function, nums is an array (Java) or tuple (Python), so you can iterate, index, etc.
Q12Analysing (Bridge)

Why does Python's flexibility with default values, keyword arguments, and *args make it easier to write than Java?

Python lets you write one function that handles multiple calling patterns. Java forces you to decide on exact parameter lists upfront and write separate methods for variations. This makes Python code shorter and more flexible, but Java code more explicit and type-safe. Trade-off: Python is easier to write, Java is easier to debug and maintain at scale.
