What Are Decorators?

A function that wraps another function to add behaviour without changing the original
›

A decorator is a function that takes another function as input, wraps it with additional behaviour, and returns the wrapper. You don't modify the original function — you layer new functionality on top of it.

Real-World Analogy

Think of a decorator like a coffee shop adding a layer of foam to a cappuccino. The coffee is still there unchanged, but now it has new behaviour (the foam). You can serve the same coffee with foam, chocolate, or cinnamon — different decorations, same underlying beverage.

In the lecture, you saw the Bottle web framework example:

@route('/hello')
def hello():
    return "Hello, World!"

The @route decorator is not Python syntax — it's syntactic sugar for hello = route('/hello')(hello). The decorator intercepts the function before it's assigned to the name hello, wraps it, and returns a new version that the web framework can manage.

🌉

Java Bridge

Java has @annotations (like @Override, @Deprecated) that look syntactically similar, but they're metadata, not function wrappers. The closest Java pattern is the Decorator design pattern — a class that wraps another class and adds behaviour. We'll see this in Step 3.

✎ Quick check
What does a decorator actually do? Does it modify the original function?
A decorator wraps the original function and returns a new function. The original function is unchanged. The wrapper function is what gets called instead. When you use the @ syntax, Python automatically reassigns the name to the wrapper, but the original function object still exists.

How Decorators Work

The wrapper pattern: outer function, inner wrapper, return the wrapper
›

Let's trace through the @do_it example from the lecture step by step.

def do_it(func):                    # Decorator: takes a function
    def wrapper(*args, **kwargs):  # Inner function: wrapper
        print("Before")
        result = func(*args, **kwargs)  # Call original function
        print("After")
        return result
    return wrapper                   # Return the wrapper

@do_it
def say_hello(name):
    print(f"Hello, {name}")

say_hello("Alice")
# Output:
# Before
# Hello, Alice
# After

Step by step:

  • do_it takes the original function as a parameter
  • Inside, we define wrapper — a new function that calls the original function with *args, **kwargs (any number of arguments)
  • wrapper adds behaviour before and after the original function runs
  • do_it returns the wrapper function, not the original
  • When you call say_hello("Alice"), you're actually calling wrapper("Alice")
💡
Universal wrapping with *args and **kwargs: The wrapper uses *args (positional arguments) and **kwargs (keyword arguments) so it can wrap any function, regardless of what parameters it takes. This is why decorators are so powerful — they work with any function.
🌉

Java Bridge

Java doesn't have function decorators (functions aren't first-class values), but the Decorator design pattern does the same thing with classes. A wrapper class implements the same interface as the original class and adds behaviour. It's less elegant but accomplishes the same goal.

✎ Quick check
Why does the wrapper use *args, **kwargs instead of specific parameters like name?
Because the decorator should work with any function, not just ones with a name parameter. *args, **kwargs capture whatever arguments the original function receives, so one decorator can wrap functions with different signatures.

Writing Your Own Decorator

Build a @timer decorator that measures execution time
›

Let's build a practical example: a decorator that measures how long a function takes to run.

import time

def timer(func):
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        elapsed = time.time() - start
        print(f"{func.__name__} took {elapsed:.4f} seconds")
        return result
    return wrapper

@timer
def slow_function():
    time.sleep(2)
    return "Done"

slow_function()
# Output: slow_function took 2.0023 seconds

Key insight: The decorator captures the original function in the closure (the wrapper function remembers func even after timer returns). Each time you call slow_function(), you're calling wrapper(), which records the timing.

✅
Real-world use: Decorators like @timer are used in production code to log function calls, measure performance, check authentication, validate input, and more. The Bottle web framework uses @route to register HTTP endpoints without you writing repetitive setup code.
✎ Quick check
If you remove the @timer decorator from slow_function, does the function still work the same way?
Yes, completely. Decorators add behaviour without changing the function's core operation. Without @timer, slow_function still runs and returns "Done" — you just don't get the timing output. That's the power of decorators: you can layer them on and off without touching the original code.

What Are Generators?

Functions that yield values one at a time instead of returning all at once
›

A generator is a function that uses yield instead of return. It doesn't compute all values at once — it pauses and resumes, producing one value at a time.

Real-World Analogy

A generator is like a bakery that makes donuts one at a time as customers order them, instead of baking all 1000 donuts before opening the shop. Each time a customer asks for a donut, the baker makes one, hands it over, pauses, and waits for the next request.

From the lecture:

def countdown(n):
    while n > 0:
        yield n          # Pause and produce a value
        n -= 1

for num in countdown(3):
    print(num)
# Output:
# 3
# 2
# 1

Each time yield runs, the function pauses. When you ask for the next value, it resumes from where it left off, executes the next iteration, and pauses again at the next yield.

🌉

Java Bridge

Java doesn't have generators, but the Iterator interface with hasNext() and next() methods does the same thing — lazily producing one value at a time. Java 8+ Streams are conceptually similar: lazy evaluation, processing one element at a time.

✎ Quick check
What's the key difference between return and yield? Why would you use a generator instead of building a list?
return exits the function and gives back a value (once). yield pauses the function and gives back a value, then resumes later. A generator is more memory-efficient than a list because it doesn't store all values at once — it computes each one on demand. For huge datasets, generators are critical.

How Generators Work

Generator objects, __next__(), StopIteration, and the resume pattern
›

When you call a generator function, you don't get a value — you get a generator object. Each call to next() on the generator runs the function until the next yield, pauses there, and returns the yielded value.

def countdown(n):
    while n > 0:
        yield n
        n -= 1

gen = countdown(3)                # Create generator object (nothing runs yet)
print(next(gen))              # Runs until first yield: prints 3
print(next(gen))              # Resumes after yield, runs loop again: prints 2
print(next(gen))              # Resumes again: prints 1
print(next(gen))              # Loop ends, raises StopIteration

The for loop automatically handles this: it calls next() repeatedly until StopIteration is raised.

💡
Generator pipelines: Generators can feed each other. One generator yields values, the next one consumes them and yields transformed values. This creates elegant, memory-efficient data processing pipelines — a common pattern in data science and large-scale text processing.
🌉

Java Bridge

In Java, you'd implement Iterator<Integer> with hasNext() and next(). The generator object is equivalent to an Iterator. StopIteration is equivalent to hasNext() returning false.

✎ Quick check
If you call countdown(5) but never call next() on the result, does the generator function run at all?
No. Calling the generator function just creates the generator object — the function body doesn't run until you call next() on it. This is why generators are so efficient: they don't do any work until you ask for a value.

Generator Expressions

Lazy list comprehensions: compute values on demand, not all at once
›

You can write a generator in one line using a generator expression — like a list comprehension but with parentheses instead of square brackets. Values are computed lazily (on demand), not all upfront.

# List comprehension — creates entire list in memory
squares_list = [x*x for x in range(1000000)]

# Generator expression — lazy: computes one value at a time
squares_gen = (x*x for x in range(1000000))

for sq in squares_gen:
    if sq > 100:
        break        # Only computed a few values — huge memory savings

Difference: The list comprehension creates all 1,000,000 squares immediately and stores them. The generator expression only computes each square when you ask for it, and stops when you break out of the loop. For large datasets, generators save massive amounts of memory.

🌉

Java Bridge

Java Streams (introduced in Java 8) have the same lazy-evaluation property: list.stream().filter(x -> x > 0).map(x -> x * x) doesn't compute all values upfront. Like generators, they're efficient for large datasets.

✎ Quick check
When would you use a generator expression over a list comprehension?
When you have a large dataset and you might not process all of it. Generators are more memory-efficient because they compute values on demand. If you need the entire result upfront, a list comprehension is fine (and slightly faster). If you're iterating once and might stop early, a generator expression is better.

Regular Expressions: The Basics

Pattern matching for text: wildcards, character classes, anchors, quantifiers
›

Regular expressions (regex) are patterns that describe sets of strings. They're used to find, match, and validate text. Python's re module provides regex support.

import re

# Find all words starting with 'a'
text = "apple, apricot, banana"
pattern = r"a\w+"              # regex pattern: 'a' followed by word chars
matches = re.findall(pattern, text)
print(matches)  # ['apple', 'apricot']

Basic patterns:

  • . — Matches any single character (except newline)
  • [abc] — Matches any one character inside the brackets
  • ^ — Start of string anchor
  • $ — End of string anchor
  • * — Zero or more of the previous character
  • + — One or more of the previous character
  • ? — Zero or one of the previous character
💡
Literal text just matches itself: The pattern "hello" matches the word "hello" anywhere in the string. Special characters like ., *, + have special meaning — if you want to match them literally, escape with \.
🌉

Java Bridge

Java has java.util.regex with the same regex language. Pattern.compile("a\\w+").matcher(text).find() — note the double backslash because Java strings also escape backslashes. The patterns themselves are identical.

✎ Quick check
What does the pattern a+ match? What about a*? How are they different?
a+ matches one or more 'a's: "a", "aa", "aaa" all match, but an empty string doesn't. a* matches zero or more 'a's: "a", "aa", and even an empty string match. The difference: + requires at least one, * allows zero.

Regular Expressions: Extended Features

Shorthand classes, repetition, capture groups, and named groups
›

Regular expressions get more powerful with shorthand classes and grouping.

# Shorthand character classes
r"\d"     # Any digit (0-9)
r"\w"     # Any word character (a-z, A-Z, 0-9, _)
r"\s"     # Any whitespace (space, tab, newline)
r"\D"     # NOT a digit

# Repetition with {m,n}
r"a{2,4}"   # 2 to 4 'a's
r"a{3}"     # Exactly 3 'a's

# Capture groups with ()
pattern = r"(\w+)@(\w+\.\w+)"   # Email: username and domain
match = re.search(pattern, "contact me at alice@example.com")
if match:
    print(match.group(0))   # Full match: alice@example.com
    print(match.group(1))   # Group 1: alice
    print(match.group(2))   # Group 2: example.com

Capture groups: Parentheses create groups that you can extract individually. group(0) is the entire match, group(1) is the first group, etc.

🌉

Java Bridge

Java uses the same shorthand classes and grouping syntax. Key difference: remember double backslashes in Java strings. Pattern.compile("\\d+").matcher(text).find() instead of re.search(r"\d+", text).

✎ Quick check
Write a regex pattern to match a 5-digit ZIP code (e.g., "12345"). Then extract the digits using a capture group.
Pattern: r"(\d{5})". To extract: match = re.search(r"(\d{5})", text); zip_code = match.group(1). The \d{5} matches exactly 5 digits, and the parentheses create a capture group so you can extract it.

Using Regex in Python

re.search(), re.findall(), re.split() — and practical parsing examples
›

Python's re module provides several functions for working with patterns.

import re

text = "Email: alice@example.com, Phone: 555-1234"

# re.search() — Find first match
match = re.search(r"(\w+)@(\w+\.\w+)", text)
if match:
    print(match.group(1))  # alice

# re.findall() — Find all matches
digits = re.findall(r"\d+", text)
print(digits)  # ['555', '1234']

# re.split() — Split by pattern
parts = re.split(r",\s*", text)  # Split on comma + optional spaces
print(parts)   # ['Email: alice@example.com', 'Phone: 555-1234']

Practical example: Parse a log file to extract user IDs and timestamps:

# Log: "User 42 logged in at 2024-01-15 10:30:45"
pattern = r"User (\d+) .* at (\d{4}-\d{2}-\d{2})"
matches = re.findall(pattern, log_line)
user_id, date = matches[0]  # ('42', '2024-01-15')
🌉

Java Bridge

Side-by-side: Python re.search() vs Java Pattern.compile().matcher().find(). The patterns are identical — the API is slightly different, but the goal is the same.

Python

match = re.search(
  r"(\d+)-(\d+)",
  "score: 42-58"
)
if match:
  print(match.group(1))
  # 42

Java

Pattern p = Pattern
  .compile("(\\d+)-(\\d+)");
Matcher m = p
  .matcher("score: 42-58");
if (m.find()) {
  System.out.println(m.group(1));
  // 42
}
✎ Quick check
How would you find all email addresses in a large text file? Which re function would you use?
Use re.findall(r"(\w+)@(\w+\.\w+)", text). findall() returns a list of all matches (or all captured groups if you have parentheses). search() only finds the first match, so findall() is the right choice here.

12 questions covering decorators, generators, regex, and the Python→Java bridge. Try to answer before revealing.
Q1Remembering

What does the @decorator syntax do? Rewrite @do_it\ndef hello(): ... without using the @ symbol.

@do_it is syntactic sugar. The equivalent without @ is: def hello(): ...; hello = do_it(hello). The decorator function takes the original function, wraps it, and reassigns the name to the wrapper.
Q2Understanding

Why do decorators use *args and **kwargs? What would happen if you hardcoded specific parameters like def wrapper(name):?

*args, **kwargs let the decorator wrap any function, regardless of what parameters it takes. If you hardcoded def wrapper(name):, the decorator would only work with functions that take a name parameter. For other functions, you'd get a TypeError.
Q3Applying

Write a @log_calls decorator that prints "Calling function_name" before each function call. Use *args, **kwargs to make it universal.

def log_calls(func):
  def wrapper(*args, **kwargs):
    print(f"Calling {func.__name__}")
    return func(*args, **kwargs)
  return wrapper
Q4Remembering

What is the difference between return and yield? Which one pauses the function?

return exits the function permanently and gives back a value. yield pauses the function, gives back a value, and resumes from that point when you ask for the next value. yield is used in generators.
Q5Understanding

When you write gen = countdown(5), what is the type of gen? Does the function body run immediately?

gen is a generator object (not an integer). The function body does not run until you call next(gen). Creating a generator is lazy — it sets up the generator object but doesn't execute any code.
Q6Applying

Write a generator function infinite_count() that yields 1, 2, 3, 4, ... forever. (Hint: use a while loop.)

def infinite_count():
  n = 1
  while True:
    yield n
    n += 1
Each call to next() increments and yields the next number. It only stops when you break out of the loop or close the generator.
Q7Understanding

When is a generator expression more memory-efficient than a list comprehension? Give an example where you'd use a generator expression.

A generator expression computes values on demand; a list comprehension creates the entire list upfront. Example: reading a million-line log file and finding errors. With gen = (line for line in logfile if 'ERROR' in line), you only process and store one matching line at a time. With a list comprehension, you'd build a huge list in memory first.
Q8Remembering

What do the regex patterns \d, \w, and \s match? Write one-character example for each.

\d = digit: "5". \w = word character (letter, digit, underscore): "A" or "_". \s = whitespace (space, tab, newline): " " (a space character).
Q9Applying

Write a regex pattern to match a phone number like "555-1234" (3 digits, hyphen, 4 digits). Then show the code to extract it from a string.

Pattern: r"(\d{3})-(\d{4})". Code:
match = re.search(r"(\d{3})-(\d{4})", text)

  area = match.group(1) # "555"

Q10Analysing (Bridge)

How does a Python decorator compare to a Java @annotation? Why doesn't Java have function decorators?

A Python decorator wraps a function to change its behaviour. A Java @annotation is metadata (tags) attached to code but doesn't wrap or change behaviour — it's just information for the compiler or runtime. Java doesn't have function decorators because functions aren't first-class values like in Python. The closest Java pattern is the Decorator design pattern (wrapping classes).
Q11Understanding (Bridge)

How would you write the Python generator countdown(3) as a Java Iterator? What methods would you need to implement?

Implement Iterator<Integer>:
public class Countdown implements Iterator<Integer> {
  private int n;
  public Countdown(int start) { this.n = start; }
  public boolean hasNext() { return n > 0; }
  public Integer next() { return n--; }
The hasNext() method replaces the generator's implicit StopIteration check.
Q12Analysing (Bridge)

Compare Python's re.findall(r"\d+", text) and Java's Pattern.compile("\\d+").matcher(text).find(). What's different about the syntax, and why?

Both do the same thing, but: (1) Python uses raw strings (r"..."); Java uses regular strings with double backslashes ("...") because backslash is an escape character in Java strings. (2) Python re.findall() is a function; Java builds a Pattern object, then a Matcher. (3) findall() returns a list of all matches; find() finds one and you call it repeatedly. The regex patterns themselves are identical.
