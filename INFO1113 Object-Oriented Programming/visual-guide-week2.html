Loops in Java: for, while, do-while

The logic is the same as Python — just different syntax
›

You know Python loops. Java has the same idea, but the syntax is different. Let's compare:

Python — range()

for i in range(5):
    print(i)

Java — C-style for loop

for(int i=0; i<5; i++) {
    System.out.println(i);
}

The Java for loop has three parts inside the brackets: initialiser (int i=0), condition (i<5), and increment (i++). They work together to repeat the block 5 times.

Initialise: i starts at 0 2 Check: is i < 5? If yes, run body 3 Increment: i++ (add 1), then loop back to 2

while loops and do-while

while loops are simpler — they just check a condition. do-while is new (Python doesn't have it): it runs the body first, then checks.

while — check first

int i = 0;
while(i < 5) {
    System.out.println(i);
    i++;
}

do-while — run first

int i = 0;
do {
    System.out.println(i);
    i++;
} while(i < 5);
🔑
Gotcha: Notice do-while ends with a semicolon after the closing }. That's because it's technically part of the condition.
✎ Quick check
If a while loop's condition is false from the start, how many times does the body run? What about a do-while?
while: 0 times — the condition is checked before the body ever runs. do-while: 1 time — the body always runs at least once because it executes before the condition check.

Enhanced For Loop (for-each)

Loop through a collection without managing an index
›

Java's enhanced for loop (called for-each) is like Python's for item in list. It iterates directly over elements:

Python

fruits = ["apple", "banana"]
for fruit in fruits:
    print(fruit)

Java — for-each

String[] fruits = {"apple", "banana"};
for(String fruit : fruits) {
    System.out.println(fruit);
}

Notice the syntax: for(Type var : array) with a colon. The variable gets each element one by one. You don't need to worry about indices.

⚠️
Important caveat: You cannot modify the array while inside a for-each loop. If you need to change elements, use a regular for loop with an index instead.
💡
Strings are tricky in for-each: You can't directly do for(char c : "hello"). Strings aren't arrays. You need for(char c : "hello".toCharArray()) — more on this in Step 5.
✎ Quick check
Write a for-each loop that prints each name from a String array called names.
for(String name : names) { System.out.println(name); } — notice the colon, and the variable type must match the array's element type.

Arrays: Declaration & Basics

Java's version of lists — but with fixed size
›

In Python, lists grow and shrink automatically. Java arrays are different: they have a fixed size, declared when you create them.

Python — grows automatically

nums = []
nums.append(5)
nums.append(10)

Java — fixed size

int[] nums = new int[5];
← declares space for exactly 5 ints

Breaking down the declaration:

int[] ← array of integers nums ← variable name new int[5] ← allocate space for 5 ints Result: a memory block with space for 5 integers, all starting at 0

Default values

When you create an array, Java fills it with defaults: 0 for int, 0.0 for double, false for boolean, and null for objects (like String).

💡
Initialising with values: You can create and fill in one go: int[] nums = {1, 2, 3, 4, 5}; — Java counts the elements and allocates the right size.
✎ Quick check
Declare a String array of 3 elements. What will be the default value of each element?
String[] arr = new String[3]; — Each element defaults to null (not an empty string, but no object at all). Trying to call methods on a null element will throw a NullPointerException.

Working With Arrays

Accessing, iterating, printing, and common patterns
›

Now you know how to declare arrays. Let's actually use them:

int[] scores = {92, 87, 95, 88};

// Access by index
System.out.println(scores[0]); ← prints 92

// Get array length
System.out.println(scores.length); ← prints 4

// Iterate with for loop
for(int i=0; i<scores.length; i++) {
    System.out.println(scores[i]);
}

Arrays.toString()

You can't just println an array directly — it prints a memory address hash like [I@15db9742. Use Arrays.toString() instead:

Wrong — memory address

System.out.println(scores);
← [I@15db9742

Right — readable

System.out.println(Arrays.toString(scores));
← [92, 87, 95, 88]
💡
You need import java.util.Arrays; at the top of your file to use Arrays.toString().

Common array patterns

Check if an array contains a value, or count occurrences. These are so common you'll write them many times:

// Does array contain a value?
boolean found = false;
for(int score : scores) {
    if(score == 95) found = true;
}

// Count occurrences
int count = 0;
for(int score : scores) {
    if(score > 90) count++;
}
System.out.println(count); ← how many scores are above 90?
✎ Quick check
Write a for loop to sum all elements in an int array called nums.
int sum = 0; for(int n : nums) { sum += n; } System.out.println(sum); — or use a regular for loop: for(int i=0; i<nums.length; i++) { sum += nums[i]; }

Strings in Java: Methods & Operations

Strings are objects with powerful built-in methods
›

You know Python strings. In Java, strings are objects, and you use methods to manipulate them. Here are the essentials:

.length()Number of characters"hello".length() → 5
.charAt(i)Get character at index i"hello".charAt(0) → 'h'
.substring(a,b)Extract part of string"hello".substring(1,4) → "ell"
.toUpperCase()Convert to uppercase"hello".toUpperCase() → "HELLO"
.toLowerCase()Convert to lowercase"HELLO".toLowerCase() → "hello"
.indexOf(str)Find position of substring"hello".indexOf("ll") → 2
.contains(str)Check if contains substring"hello".contains("ll") → true

Important difference from Python

In Python, you use len(s) to get string length. In Java, it's the method .length() (with parentheses). Also, .substring(a,b) doesn't include the end index — like Python slicing.

✎ Quick check
Given String name = "Eduardo";, write code to get the first three characters.
name.substring(0, 3) → "Edu" (note: the end index 3 is exclusive, so you get indices 0, 1, 2)

String Comparison Trap: == vs .equals()

The classic Java gotcha — this catches everyone
›

This is a critical concept. Watch what happens when you compare strings:

WRONG — using ==

String a = "hello";
String b = "hello";
if(a == b) { // Sometimes true!
    System.out.println("Equal");
}

RIGHT — using .equals()

String a = "hello";
String b = "hello";
if(a.equals(b)) { // Always true
    System.out.println("Equal");
}

Why? Because == checks if two variables point to the exact same object in memory. .equals() checks if the content is the same. Here's a memory diagram:

a == b? FALSE (different objects) a.equals(b)? TRUE (same content)

The same string content can exist in two different places in memory. == checks the memory address. .equals() checks the content. Always use .equals() for string comparison.

⚠️
Case-insensitive comparison: Use .equalsIgnoreCase(other) to ignore case: "hello".equalsIgnoreCase("HELLO") → true.
✎ Quick check
Why is if(userInput == "hello") a bug? What should you use instead?
== checks if userInput and "hello" are the exact same object in memory — they almost certainly aren't, even if they have the same text. Use userInput.equals("hello") to check if the content is the same.

Command-Line Arguments

Passing information to your program when you run it
›

Remember in Python lesson 8, you learned sys.argv? Java has the same concept built into main. Those String[] args parameters capture command-line arguments:

public static void main(String[] args) {
    // args[0] is the first argument passed on command line
    // args[1] is the second, etc.
    // args.length is the count
}

For example, if you run:

java MyProgram hello 42 world

Then inside main:

args[0]"hello"
args[1]"42" (note: it's a String, not an int!)
args[2]"world"
args.length3
💡
Arguments are always Strings. If you need a number, convert it: int age = Integer.parseInt(args[0]);.
⚠️
Different from Python! In Python's sys.argv, argv[0] is the program name. In Java, args[0] is the first argument — the program name is not included.
✎ Quick check
Write code to read the first command-line argument and print it in uppercase.
if(args.length > 0) { System.out.println(args[0].toUpperCase()); } — always check the length first to avoid an ArrayIndexOutOfBoundsException.

Static Methods as Utilities

Helper functions that don't need object data
›

You learned about static vs instance methods in Week 1. This week, you'll write static methods as utility functions — tools that work with arrays and strings:

public static boolean contains(int[] arr, int target) {
    for(int x : arr) {
        if(x == target) return true;
    }
    return false;
}

public static int count(int[] arr, int target) {
    int c = 0;
    for(int x : arr) {
        if(x == target) c++;
    }
    return c;
}

Why are these static? Because all the data comes through parameters. They don't depend on any object's fields. You call them like this:

int[] nums = {5, 10, 5, 15};
if(contains(nums, 10)) {
    System.out.println("Found it!");
}
System.out.println(count(nums, 5)); ← prints 2

Decision: static or instance?

Make it static if:

• All data comes from parameters
• It doesn't need this
• Example: contains(arr, val)
  No object data needed.

Make it instance if:

• It uses the object's fields
• Example: a Car object's
  getSpeed() method reads
  this.currentSpeed
✎ Quick check
Write a static method called max that takes an int array and returns the largest element.
public static int max(int[] arr) { int m = arr[0]; for(int x : arr) { if(x > m) m = x; } return m; } — Note: this assumes arr is not empty.

Putting It Together: Occurrence Counter

A real example tying loops, arrays, strings, and static methods
›

Here's a program from the course that demonstrates everything you've learned this week. It counts how many times each digit appears in a string:

public class Occurrence {
    public static void main(String[] args) {
        String input = args[0]; // Get from command line

        // Create array to count digits 0-9
        int[] counts = new int[10];

        // Loop through each character in the string
        for(int i = 0; i < input.length(); i++) {
            char ch = input.charAt(i);

            // If it's a digit, increment its counter
            if(ch >= '0' && ch <= '9') {
                int digit = ch - '0'; // Convert char to int
                counts[digit]++;
            }
        }

        // Print results
        for(int d = 0; d < 10; d++) {
            if(counts[d] > 0) {
                System.out.println("Digit " + d + ": " + counts[d]);
            }
        }
    }
}

Run it: java Occurrence 3141592653 — Output shows how many times each digit appears.

Breaking it down

  • Command-line arg: args[0] captures the input string
  • Array: counts[10] — one slot for each digit
  • Loop: for(int i = 0; i < input.length(); i++) — iterate through string
  • String method: charAt(i) — get each character
  • Char to int: ch - '0' — clever trick to convert '3' to 3
  • Array indexing: counts[digit]++ — increment the right counter
✎ Quick check
What would happen if you removed the if(counts[d] > 0) check in the output loop?
It would print all digits 0-9, even those with count 0. The output would be messier, showing "Digit 4: 0" for digits that never appeared.

2D Arrays (Brief Intro)

Arrays of arrays — grids of data
›

You know regular arrays. A 2D array is an array of arrays — think of it as a grid or table:

Declare a 2D array

int[][] grid =
    new int[3][4];
← 3 rows, 4 columns

Access elements

grid[0][0] = 5;
grid[1][2] = 10;
System.out.println(grid[0][1]);

To traverse a 2D array, you nest loops:

for(int row = 0; row < 3; row++) {
    for(int col = 0; col < 4; col++) {
        System.out.print(grid[row][col] + " ");
    }
    System.out.println(); // New line after each row
}

Python connection

In Python, you might have used nested lists: grid = [[1, 2], [3, 4]]. Java 2D arrays are similar, but with fixed dimensions. Each row must have the same number of columns (in a regular 2D array).

💡
Get dimensions: grid.length is the number of rows. grid[0].length is the number of columns (of the first row).
✎ Quick check
Declare a 2D array called matrix with 2 rows and 3 columns, and initialise it with the values 1, 2, 3 in the first row and 4, 5, 6 in the second.
int[][] matrix = {{1, 2, 3}, {4, 5, 6}}; or int[][] matrix = new int[2][3]; matrix[0][0]=1; matrix[0][1]=2; etc.

Q2Understanding

What is the key difference between a while loop and a do-while loop? When would you choose one over the other?

A while loop checks the condition before executing the body — it might never run. A do-while always executes at least once, then checks. Use do-while when you need the body to run at minimum once (e.g., menu prompt, input validation).
Q3Applying

Q4Understanding

Explain what happens when you try to access an array index that doesn't exist.

Java throws an ArrayIndexOutOfBoundsException at runtime. Unlike Python (which uses negative indices for wrap-around), Java has no negative indexing — the valid range is 0 to array.length-1.
Q5Applying

Write a static method called countOccurrences(int[] arr, int target) that returns how many times target appears in the array.

public static int countOccurrences(int[] arr, int target) { int count=0; for(int x:arr) if(x==target) count++; return count; }
Q6Understanding

Why must you use .equals() instead of == to compare Strings in Java?

== compares references (memory addresses) — are these the exact same object? .equals() compares content — do they contain the same characters? Two different String objects can have the same text but == would return false.
Q7Remember

What does Arrays.toString(arr) do and why is it necessary?

Arrays.toString() returns a human-readable string representation like "[1, 2, 3]". Without it, printing an array with System.out.println gives the memory address hash (e.g., [I@15db9742) because arrays don't override toString().

Explain how command-line arguments work in Java. How do you access them?
