First Impressions: Why Does Java Look Like That?

Don't panic — there's a reason for all the extra words
›

Here's the simplest possible program in Python, and the same thing in Java:

Python — 1 line

print("Hello!")

Java — 5 lines

public class Hello {
    public static void main(String[] args) {
        System.out.println("Hello!");
    }
}

Yes, Java is more verbose. That's not a mistake — it's a design choice. Every bit of that "extra stuff" exists for a reason, and by the end of this guide you'll understand each piece.

💡
The main method is the entry point. When you run a Java program, the JVM (Java Virtual Machine) looks for a method with this exact signature: public static void main(String[] args). If it can't find one, your program won't start. Every Java program you run this semester needs this.
👍
You don't need to memorise this yet. Right now, just understand the nesting: your code goes inside main, which goes inside a class. That's the sandwich. You'll type it so many times it becomes automatic.
✎ Quick check
In your own words: what are the two "wrappers" that surround your actual code in a Java program? Which one does Java look for first when starting?
The outer wrapper is a class (public class Hello { }). The inner wrapper is the main method (public static void main(String[] args) { }). Your instructions go inside main, which goes inside the class. When you run the program, Java looks for main — that's where execution begins.

Compile, Then Run: Java's Two-Step Process

Python runs directly — Java checks your work first
›

In Python, you type python hello.py and it runs. Java has an extra step: it compiles your code first, checking for errors before anything runs.

Key terms

Bytecode is the compiled .class file — not human-readable, but what the JVM understands. The JVM (Java Virtual Machine) is the program that runs your bytecode. This two-step process is why Java catches many errors before your program ever runs.

💡
Why is this useful? In Python, a typo on line 50 only crashes when your program reaches that line. Java's compiler checks everything before running — it catches mistakes up front.
⚠️
The file name must match the class name — exactly. If your class is called Hello, the file must be Hello.java. Capital H and all. This catches everyone at least once.
# In the terminal:
javac Hello.java     ← compile (creates Hello.class)
java Hello           ← run (note: NO .java or .class extension)
✎ Quick check
Your file is called MyProgram.java. What two commands do you type to compile and run it?
javac MyProgram.java to compile, then java MyProgram to run. You type the .java extension for compiling but not when running. The class inside must also be called MyProgram — mismatch = compiler error.

Every Variable Has a Type

Java won't let you put text where a number should be
›

In Python, you write x = 5 and Python figures out the type. Java makes you say the type out loud. This is called strong typing — once a variable has a type, it can never hold a different type.

Python — type is automatic

age = 19
name = "Will"
price = 4.50
passed = True

Java — you declare the type

int age = 19;
String name = "Will";
double price = 4.50;
boolean passed = true;

The types you'll use most:

intWhole numbersint age = 19;
doubleDecimal numbersdouble price = 4.50;
StringText (note the capital S)String name = "Will";
booleantrue or falseboolean ok = true;
charSingle characterchar grade = 'A';
voidNo value (for methods that don't return anything)public void sayHello() { ... }
⚠️
Every line ends with a semicolon ; — Java requires this. Python doesn't. You will forget semicolons. Also: Java's true/false are lowercase, unlike Python's True/False.
💡
What's void? You saw public static void main(...) in Step 1. void means "this method doesn't give back a value." It does its job (printing, for example) but doesn't return a result. Methods that return int, String, etc. give back a value you can store.
✎ Quick check
How would you declare a variable called temperature that holds the value 36.5 in Java?
double temperature = 36.5; — double because it's a decimal, and don't forget the semicolon.

Getting Input & Importing Tools

Java's version of Python's input() — plus how packages work
›

In Python, getting input is one line: name = input("Name: "). In Java, you need a Scanner — a special tool that reads from the keyboard — and you have to import it first.

Python — built in

name = input("Name: ")
print("Hello,", name)

Java — need Scanner

import java.util.Scanner;

public class Greet {
  public static void main(String[] args) {
    Scanner scan = new Scanner(System.in);
    System.out.print("Name: ");
    String name = scan.nextLine();
    System.out.println("Hello, " + name);
  }
}

Key details

import java.util.Scanner; — goes at the very top of the file. new Scanner(System.in) — creates a scanner listening to keyboard. scan.nextLine() reads text, scan.nextInt() reads a whole number, scan.nextDouble() reads a decimal.

💡
Imports and Packages: Java organises tools into packages — like folders of useful classes. java.util contains Scanner, ArrayList, and more. The import line tells Java "I want to use this tool." Some things like System.out.println and String don't need importing — they're in java.lang, which is always available.
💡
print vs println: System.out.print() stays on the same line. System.out.println() adds a new line at the end. Use print for prompts and println for output.
✎ Quick check
You've created a Scanner called scan. What line reads a whole number into a variable called age?
int age = scan.nextInt(); — type (int), variable name (age), scanner method (nextInt()), semicolon.

Scope: What the Curly Braces Mean

In Python it's indentation — in Java it's { and }
›

In Python, indentation defines what belongs to what. In Java, curly braces { } do that job. Everything between { and } is one block, and variables created inside a block only exist inside that block. That's called scope.

Python — indentation = scope

if age >= 18:
    msg = "Welcome"
    print(msg)
# msg still exists here

Java — curly braces = scope

if (age >= 18) {
    String msg = "Welcome";
    System.out.println(msg);
}
// msg does NOT exist here!
⚠️
Common trap: declaring a variable inside an if or for block and then trying to use it afterwards. Java won't compile. If you need a variable after the block, declare it before the opening {.
💡
Nesting matters: class braces contain method braces, which contain code. Every { must have a matching }. A missing } is one of the most common beginner errors.
✎ Quick check
You declare int count = 0; inside a for loop's braces. Can you use count after the loop? Why or why not?
No. A variable declared inside { } only exists within those braces — that's its scope. Once the closing } is reached, the variable is gone. Declare it before the loop if you need it afterwards.

What is Object-Oriented Programming?

The big idea behind this entire course
›

Up until now, you've been writing code as a sequence of steps — do this, then this, then this. That's called procedural programming. All your Python code has been procedural. It works fine for small programs.

Object-Oriented Programming (OOP) is a different way of organising code. Instead of writing steps, you create things — objects — that each know how to do their own job. OOP is a paradigm (a style of thinking), not a language. Java was built for it.

Your lecturer's example: The BouncerBot

Imagine a nightclub hires a robot bouncer. In procedural code, you'd write all the logic as one long sequence. In OOP, you'd create separate things: a Person (name and age), a BouncerBot (knows the rules), and a Club (puts them together). Each one is responsible for its own job.

🔑
The key shift: instead of writing instructions that do things to data, you create objects that contain data and know what to do with it.
✎ Quick check
In the BouncerBot example, why is "minimum age" stored in the BouncerBot rather than in the Club or the Person?
Because checking age is the bouncer's job. The BouncerBot needs to know the rule, so it stores the minimum age. Each object is responsible for the data it needs to do its job.

Classes and Objects: Blueprint vs. Real Thing

This is the single most important idea in Java
›
Think of it this way

A class is like a form template — "Person: name ___, age ___". An object is a filled-in form — "Person: name Will, age 19". One template, unlimited forms.

A class has two kinds of things: fields (data it stores — also called instance variables) and methods (actions it can do). Fields are the blank spaces on the form; methods are the instructions on the form.

// The CLASS — the template
public class Person {
    String name;     // field
    int age;         // field
}

// Creating OBJECTS — filling in the template
Person lecturer = new Person();
lecturer.name = "Eduardo";
lecturer.age = 38;

Person student = new Person();
student.name = "Will";
student.age = 19;

The keyword new creates an object from the class. Without new, you're just declaring a variable — nothing exists in memory yet.

✎ Quick check
What's the difference between a class and an object? Give a concrete example (not Person).
A class is a template — defines what data and actions something has. An object is a specific instance with actual values. Example: Car is a class (colour, speed, drive()). new Car("red", 60) is an object — a specific car. You can create many objects from one class.

Constructors: Setting Up an Object When It's Created

Instead of setting fields one by one, do it all at once
›

In Step 7, we set name and age separately after creating the object. That's clumsy — what if you forget one? A constructor is a special method that runs automatically when you say new, letting you provide all the data up front.

Without constructor (messy)

Person p = new Person();
p.name = "Will";  // easy to forget
p.age = 19;

With constructor (clean)

Person p = new Person("Will", 19);
// name and age set immediately
public class Person {
    String name;
    int age;

    // Constructor — same name as the class, no return type
    public Person(String name, int age) {
        this.name = name;   // this.name = the FIELD
        this.age = age;     // name = the PARAMETER
    }
}

The tricky bit: this

The parameter and the field are both called name. this.name means "the field on this object." Without this, name = name; assigns the parameter to itself — the field stays empty.

✎ Quick check
Write a constructor for a Car class that takes String colour and int wheels. Use this to set the fields.
public Car(String colour, int wheels) { this.colour = colour; this.wheels = wheels; } — same name as the class, no return type, this to distinguish fields from parameters.

Encapsulation: Locking the Data, Providing a Window

Why Java makes you hide things and use getters/setters
›
Think of it this way

A car has pedals and a steering wheel (public interface) but you can't reach inside and rearrange the engine (private data). That's encapsulation: the object controls who can see and change its data.

Mark fields as private so nothing outside the class can touch them. Then provide public getter methods (for reading) and optionally setter methods (for writing).

public class Person {
    private String name;    // locked
    private int age;

    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }

    // Getters — read access
    public String getName() { return this.name; }
    public int getAge() { return this.age; }

    // Setter — write access (only add if you WANT outside code to change it)
    public void setName(String name) { this.name = name; }
    // No setAge — age can't be changed after creation!
}

Why bother?

If age were public, any code could set person.age = -5. With private fields and no setter, age is set once in the constructor and stays locked. Setters are optional — only add them when you want controlled write access.

✎ Quick check
A field is private int score; and there's no setter. Can code outside the class change the score?
No. private means only the class itself can access the field. Without a setter, there's no way for outside code to change it. If there's a getter, outside code can read but not write — read-only access.

Static vs Instance: Does It Need an Object?

One simple question decides which to use
›

Ask one question: "Does this method need to know about a specific object's data?"

STATIC (no object needed)INSTANCE (needs a specific object)
All data comes through parametersReads data from this object
static double area(double w, double h)String getName()
Call on the class: Math.max(5, 10)Call on an object: p.getName()
Like a calculator — give it numbers, get a resultLike asking a person — "what's YOUR name?"
💡
Why main is static: when your program starts, no objects exist yet. The JVM needs a method it can call without creating an object first. That's public static void main.
✎ Quick check
Static or instance? (a) computeArea(double width, double height) (b) getSpeed() on a Car object — no parameters
(a) Static — all data comes through parameters. (b) Instance — no parameters means the data comes from this object. It needs to know which car's speed.

Single Responsibility: Each Class Does One Job

The design rule that makes OOP actually work
›

Now that you know how to create classes, there's one important design rule: the Single Responsibility Principle (SRP). Each class should have one job — not two, not five.

Think of it this way

In a restaurant, the chef cooks, the waiter serves, and the cashier handles payment. You wouldn't ask the chef to also collect money. Each person has a clear role. Same with classes.

Bad: one class does everything

public class DoEverything {
  String name; int age; int minAge;
  // stores data AND checks rules
  // AND runs the program
  // hard to change, hard to test
}

Good: each class has one job

Person     → stores identity
BouncerBot → checks ages
Club       → runs the show
// each is small, focused, testable

Why it matters

When each class has one job, bugs are easy to find — if age-checking is broken, look at BouncerBot. You can also change one class without breaking the others. Your lecturer will expect this in assignments.

✎ Quick check
You're building a library system. Should a Book class also handle checking books out to members?
No. A Book stores book data (title, author, ISBN). Checkout logic belongs in a separate class like Library or CheckoutSystem. Each class does one thing well — that's SRP.

Putting It All Together: The BouncerBot

Everything from Steps 1–11 in one real program
›

From your lecture. Three classes, each with a single responsibility:

Person.java — stores data, provides getters (encapsulation)

public class Person {
    private String name;
    private int age;

    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }

    public String getName() { return this.name; }
    public int getAge() { return this.age; }
}

BouncerBot.java — knows the rules, checks a Person

public class BouncerBot {
    private int minimumAge;

    public BouncerBot(int minimumAge) {
        this.minimumAge = minimumAge;
    }

    public void checkAge(Person p) {
        if (p.getAge() >= this.minimumAge) {
            System.out.println(p.getName() + ", welcome in!");
        } else {
            System.out.println(p.getName() + ", sorry, too young.");
        }
    }
}

Club.java — the main program (entry point)

public class Club {
    public static void main(String[] args) {
        Person eduardo = new Person("Eduardo", 38);
        Person will = new Person("Will", 19);
        Person kid = new Person("Timmy", 12);

        BouncerBot bouncer = new BouncerBot(18);

        bouncer.checkAge(eduardo);  // Eduardo, welcome in!
        bouncer.checkAge(will);     // Will, welcome in!
        bouncer.checkAge(kid);      // Timmy, sorry, too young.
    }
}
🔑
Notice: Club doesn't check ages itself — it asks the bouncer. BouncerBot doesn't know names — it asks the Person. Each object does its own job. That's OOP and the Single Responsibility Principle in action.
✎ Quick check
Why does bouncer.checkAge(will) know Will's name and age, even though we only passed the Person object?
The will object carries its own data (name="Will", age=19). Inside checkAge, the BouncerBot calls p.getName() and p.getAge() to ask the Person for its data. The object brings everything with it.

Week 1 in one sentence

Java organises code into classes (templates) that create objects (real things with their own data), hides data with private fields and gives controlled access through public getters/setters, uses constructors to set up objects, and gives each class a single responsibility.

📝 Week 1 Self-Test
10 exam-style questions. Try to answer each one before revealing. These cover the full study guide.
Q1Understanding

What is the difference between a class and an object? Give a concrete example.

A class is a blueprint defining fields and methods. An object is a specific instance with actual values. Example: Car class defines colour and speed. new Car("red", 60) is a specific Car object.
Q2Applying

Write the constructor syntax for a Car class that takes String colour and int wheels.

public Car(String colour, int wheels) { this.colour = colour; this.wheels = wheels; }
Q3Understanding

If I make a field private and don't write a setter, what does that mean for code outside the class?

Code outside the class cannot change that field. It's read-only (if a getter exists) or completely invisible (if no getter either). Set once in the constructor, locked forever.
Q4Analysing

Should computeArea(double width, double height) be static or non-static? Why?

Static. All data comes through parameters — it doesn't need any object's fields. Like a calculator function.
Q5Analysing

Should getSpeed() (no parameters) on a Car object be static or non-static? Why?

Non-static (instance). No parameters means the data comes from this object's speed field. It needs to know which car.
Q6Understanding

What does this.age = age; mean inside a constructor? What if you wrote age = age; instead?

this.age = the field. age = the parameter. So it sets the field to the parameter value. Without this, age = age; assigns the parameter to itself — the field stays at 0.
Q7Understanding

What happens if your file is Bouncer.java but the class inside is BouncerBot?

Compiler error. The file name must match the public class name exactly (including capitalisation).
Q8Understanding

Explain the Java compilation pipeline: what are the three steps from source code to running program?

1. Write source code (.java file). 2. Compile with javac → produces bytecode (.class file). Errors stop here. 3. Run with java → JVM executes the bytecode.
Q9Understanding

Why is encapsulation described as being like "a car with pedals and steering wheel but a hidden engine"?

Pedals/steering = public methods (the interface). Engine = private data (hidden internals). You interact through controls, not by reaching inside. The class manages its own state.
Q10Applying

A Student class has private String name and private double gpa. Write the getter methods for both.

public String getName() { return this.name; } and public double getGpa() { return this.gpa; }
